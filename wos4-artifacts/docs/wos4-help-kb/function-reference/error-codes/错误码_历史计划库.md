# 1.1 库服务启动异常

### 1.1 库服务启动异常

# 1.1.1 Case1

问题描述：
​ 停止历史库服务后，再重启历史库服务，历史库服务实例运行状态为“已运行”，但是历史库的数据区运行状态为“不可用”。
问题原因：
​ 启动历史库服务，有一个加载回放数据到内存的过程，当历史库里的对象数量较多时，耗时较长，需等一段时间。如果数据区一直不可用，并且历史库有崩溃或重启的日志，可能是内存不足导致的，检查一下内存占用情况。
