# HttpVersion（http版本）

"1.0"
1.0版本
server采用同步架构处理请求，请求线程会卡着，一直等待server的响应。如果响应超时到，FU还没有回传数据，接口库系统内部会自动回传1个statusCode=500的响应数据，结束等待。默认短连接。
"1.1"
1.1版本
server采用同步架构处理请求，长连接。
"2.0"
2.0版本
server采用异步架构处理请求，请求线程不会等待server的响应，不会卡着，避免队头堵塞。长连接，多工、多路复用。

# Status（通信链路状态）

// 通信链路状态
Enum Enum_IF_Status
{
 NORMAL_DEFAULT = 0, // 服务未打开/客户端未打开
 NORMAL_OPEN = 1, // 服务正常开启/客户端正常连接; 用户打开接口后，连接正常运行
 NORMAL_CLOSE = 2, // 服务正常关闭/客户端正常断开; 用户未打开接口或用户主动关闭接口
 ERROR_CLOSE = 3, // 服务异常停止/客户端异常断开; 用户打开接口后，连接系统主动断开、心跳检测失败不可用等系统行为导致断开
 ERROR_OPEN = 4, // 服务异常运行/客户端异常连接; 用户关闭接口后，连接由于系统原因依旧在运行，没有断开成功
};
