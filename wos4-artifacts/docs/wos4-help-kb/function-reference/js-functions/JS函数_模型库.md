# 2.1. 入参规则

1、param、input中，必填字段缺少时报错，填写多余字段时忽略。
2、写操作中，给不存在的字段赋值报错。
3、读操作中，查询不存在的字段不报错，返回为null。
4、函数入参的字段名和数据结构的属性名都是不区分大小写的。
5、数值类型参数赋值是支持匹配转换的，比如int32类型可以赋值int16或int64的值。

## 2.3. 错误码规则

# 2.3.1. 返回码和错误码返回规则

1、所有函数中，如果input下一级参数（如input.createMode）存在错误，ret返回具体错误原因，errorCodes为空。
2、input下一级参数全部正确的情况下，errorCodes长度一定与二级参数数组（如input.ids数组）长度对应。
3、如果二级参数在参数校验时存在错误，需继续校验入参，最终ret返回未执行，errorCodes返回未执行和错误码，如【-10003，-10021，-10003，-10022，-10003】
4、如果二级参数在执行时导致错误，写操作中止执行，最终ret返回全部失败，errorCodes返回成功、未执行和错误码，如【0，-10021，-10003，-10003，-10003】
5、如果二级参数在执行时导致错误，读操作继续执行，最终ret返回成功，errorCodes返回成功和错误码，如【0，-10021，0，0，0】，dataset返回读到的结果。dataset行数与入参一致。

# 2.3.3. 函数入参错误校验规则

用户入参存在明显错误时，需要直接返回错误，函数不执行，以此减少执行的时间，并节省系统性能损耗。
用户入参出现多处错误时，需要一次全部校验完成，达到一次试错一次修改即可执行成功的目的。
校验规则包括但不限于如下：
1、函数的各个入参stringMap中，校验是否缺少必填项。例如Create函数创建成员是，dataValues中缺少name属性。
2、操作对象选择路径时，校验路径是否存在。例如查询指定仓库下的指定名称的对象，指定的仓库不存在。
3、批量操作时的入参数组，校验数组元素数量是否在产品定义范围内。例如批量添加成员超过1024个。
4、string类型的属性赋值，校验string长度是否在产品定义范围内。例如string类型成员赋值超过1024字符。
5、对象固有属性赋值，校验属性的值类型与用户入参的数据类型是否匹配。例如给string类型的仓库GUID赋值uint64类型值。
