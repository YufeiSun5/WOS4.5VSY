# 主客体访问范畴

主体和客体可以建立对应的细分类，用来应对更细粒度权限的控制。
例如，两个车间内有分别有属于各自车间的物料，不同车间内的工人只能访问各自车间内的物料信息，此时，就可以通过以下步骤来完成权限的控制：
给【数据库用户】创建两个细分类，分别为车间一用户和车间二用户；

给物料对象【实时对象】创建两个细分类，分别为车间一物料（数据对象）和车间二物料（数据对象）；

给车间一的用户和车间二的用户配置所属类别；

给车间一的物料和车间二的物料配置所属类别；

配置访问关系。

# 1.细分类创建

选择左侧具体的对象，以业务用户为例，点击【新建】按钮，进行细分类的创建。
![image-20240531172415492](media/image-20240531172415492.png)

## 1.细分类创建

 选择左侧具体的对象，以业务用户为例，点击【新建】按钮，进行细分类的创建。

 

# 2.细分类配置

建了对应类别的细分类后，可以修改对象的所属类别。这里依然以数据库用户为例，具体步骤为：
选择具体的用户，点击【修改安全标记】按钮；

修改安全类别属性。

![image-20240531173356621](media/image-20240531173356621.png)

## 2.细分类配置

 建了对应类别的细分类后，可以修改对象的所属类别。这里依然以数据库用户为例，具体步骤为：

 
 选择具体的用户，点击【修改安全标记】按钮；

 修改安全类别属性。

 

 

# 3.访问范围配置

主体能访问哪些类别的型（包括细分型），需要通过访问范围配置来完成。
![image-20240531173909576](media/image-20240531173909576.png)
鼠标切换到【主客体操作关系】，选择左侧具体的对象，以业务用户为例，点击【修改】按钮进行访问关系配置。
![image-20240531173839274](media/image-20240531173839274.png)
修改访问关系时，需要说明一点的是，域拥有了某个型的访问权限，就默认拥有了这个型对应的所有细分型访问权限。

## 3.访问范围配置

 主体能访问哪些类别的型（包括细分型），需要通过访问范围配置来完成。

 

 鼠标切换到【主客体操作关系】，选择左侧具体的对象，以业务用户为例，点击【修改】按钮进行访问关系配置。

 

 修改访问关系时，需要说明一点的是，域拥有了某个型的访问权限，就默认拥有了这个型对应的所有细分型访问权限。
