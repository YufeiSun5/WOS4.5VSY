# 时空库

WellinOS提供将客观世界万事万物快速转化为数字孪生体的“数字孪生”方法，并提供管理数字孪生体全生命周期的功能。
WellinOS通过管理数据对象的三种时态对象：实时对象、历史对象、计划对象，实现对数字孪生体的全生命周期的管理功能。
实时库是存储和管理实时对象的数据库，部分实时对象还包括数据记录。实时库提供实时对象、记录的存储、修改、查询、订阅等功能。用户可以通过App访问实时库中的实时对象及其记录。
历史库是存储和管理历史对象、以及历史对象数据记录的数据库，历史库提供历史对象、记录的存储、修改、查询、订阅等功能。用户可以通过App访问历史库中的历史对象及其记录。
计划库是存储和管理历史对象、以及计划对象数据记录的数据库，计划库提供计划对象、记录的存储、修改、查询、订阅等功能。用户可以通过App访问计划库中的历史对象及其记录。
在时空库中，物是客观世界中物理实体的数字孪生体，事是客观世界中的事件的数字孪生体。按变化和存储频率，又分为业务数据、时序数据。
业务物、业务事的变化频率较低，其数据不需要频繁写库。时序物、时序事的变化频率高，其数据需要频繁地写库。
例如：汽车、轮胎是业务物，订单、工单是业务事，设备报警是时序事。
单值频繁写库是单灵敏时序物，比如传感器。多值一同频繁写库是多灵敏时序物，比如采集设备（如定时采集温度、压力、流量等多个值）。

# 时序物

时序物主要指变化较频繁的物，其数据需要频繁地写库，时序物有自己独特的属性：新鲜期（实时）、有效期（历史），单灵敏度时序物还有变化死区（实时）、存储死区（历史）等。
时序物：分为单灵敏度时序物、多灵敏度时序物。
单灵敏度时序物：单点时序数据，数量比较固定，变化快，刷新频率快，例如：单个传感器。刷新频率高，对网络带宽要求高，计算速度快、对CPU要求高。
多灵敏度时序物：结构化时序数据，数量固定，变化快，刷新频率快，时间戳要对齐，例如：设备仪表，电表（电量、电压、功率）。
单值频繁写库使用单灵敏度时序物，比如传感器。
多值一同频繁写库使用多灵敏度时序物，比如采集设备（如定时采集温度、压力、流量等多个值）。
单灵敏度时序物分为实时单灵敏度时序物和历史单灵敏度时序物，没有计划单灵敏度时序物。
多灵敏度时序物分为实时多灵敏度时序物和历史多灵敏度时序物，没有计划多灵敏度时序物。
历史时序物对象有历史记录，实时时序物没有记录，实时时序物对象存于内存之中，以应对较频繁的变化。

# 时空对象

时空对象是业务单元的运行容器，时空之间彼此隔离，容纳不同业务模块的程序和数据。
时空对象在WOS云中具有唯一性（同云同数据区下不能重名），时空对象有父子关系，是一个时空树。
系统时空的ID固定为0，系统时空本身不可删除、修改、查询。但可以作为范围查询包含的对象。
创建子时空时需要指定父时空，创建顶层时空时指定的父时空是系统时空0。同层级的时空数量数量不做限制。时空层级数量不做限制。 
对于非部署数据区，直接在运行库创建时空对象。
对于部署数据区，要先在模型库创建时空对象，再部署到运行库。

# 时空约束

App对象写（增、删、改）本云、非本云的数据（对象、对象的记录）时，受安全权限+App对象时空（this.AppSpaceTimeGUID）的限制，只能写本App对象时空内（不包括后代时空、兄弟时空、上级时空）的数据，在指定云/指定数据区/本App对象时空范围里写数据（对象、对象的记录）。
时空约束可以促使用户开发项目应用时，让业务模块功能内聚、独立，减少模块之间的耦合影响，方便维护。

# 时空筛选

App对象读（查询、订阅）本云、非本云数据时，只受安全权限限制，不受App对象时空的限制，可以设置input.spaceTimeID参数，在指定云/指定数据区/指定时空里查询数据，如果没有设置input.spaceTimeID，则在指定云/指定数据区/所有时空里查询数据。
时空筛选模式有3种：本App时空、指定时空、数据区所有时空。
spaceTimeMode=1、省略：表示按本App时空筛选。
spaceTimeMode=2：表示按指定时空筛选，指定时空参数要必填。
spaceTimeMode=3：表示按数据区所有时空筛选。

# 重名

时空对象的名称，在其父时空下唯一。时空对象的别名，数据区内唯一。
数据对象的名称，在时空下不唯一，同一AU下要唯一。
同一时空同一AU内的同类型对象不能重名。非部署模式数据区的时空下只有一个虚拟的系统AU，所以，非部署模式数据区的每个时空里的同类型对象都不能重名。
数据对象的别名，时空内唯一。
根据指定类型，按名称、别名操作指定数据对象时（mode=13、17），如果发现有重名对象，按duplicationDealMode处理，其他查询模式，比如父子查询、祖先后代查询等发现重名，按失败报错处理；增、删、改时，如果发现重名对象，按失败报错处理。

# 有效期validPeriod

有效期，是历史库的单灵敏时序物对象和多灵敏时序物对象的特有属性。有效期的单位为毫秒ms。
工业实时数据对于“真实性”要求往往非常高，因此，实时时序物对象具有新鲜期、是否新鲜的属性，如果时序物的值不新鲜了，就表明该值是不可靠的。
实时库时序物的值的更新时间和查询时间的时间差如果超过新鲜期，就是不新鲜的，如果在新鲜期内，就是新鲜的。新鲜期为0表示永远新鲜。
历史库中，与新鲜期相似的属性是有效期，在添加时序物历史记录时，有效期与存储死区共同作用。
生产中的数据，都是在一段时间范围内有效的，超过这个时间范围，对业务或生产就失去了价值，可能会造成巨大的损失。如养殖场自动饮水系统，水槽高度低于指定下限值，会产生报警，需要及时进行补水，如果不补水，现有水量仅能维持一天正常使用，那么这条水槽高度数据的有效期是1天，超过1天，可能会造成生命的代价。行驶中的汽车，油箱液位报警，这条报警记录，在补充加油之前都是有效的，提醒驾驶员及时加油；如果油耗尽了，还没找到加油站加油，那么这条记录报警记录已经没有提醒的价值了。
从添加的时序物记录的时间戳[假设为t1]开始，如果没有有效期内（时间戳大于t1、小于等于t1+有效期）的记录添加进来，则在t1+有效期的时间上，系统自动插入一条无效即质量为250的记录。（系统定期补）（视情况实现）。

时序物记录的统计计算功能，统计的都是原始记录，不统计质量戳为250的记录，不统计有损压缩掉的记录，有损压缩包括死区压缩和旋转门压缩。

# 显示小数位数约束displayDigits

displayDigits默认值为20，20表示不限制小数位数，存的多少位，就返回多少位。
displayDigits>=0时：表示显示时保留小数位数。显示小数位数对值存储不起作用，只对查询返回起作用，查询时，浮点型成员按照显示小数位数配置进行四舍五入后返回，如果存储的数值的小数位数小于设置的显示位数，则该设置不起作用，存的是什么就返回什么。

minValue、maxValue：最小值、最大值约束，是设置成员的最小值、最大值。对数值类型（整型、浮点、decimal、时间）的成员有效，对非数值类型（bool、string、blob）的成员无效，minValue、maxValue赋值时需要和成员类型保持一致。

# 运行模型的成员索引

运行模型成员索引，是配置在运行模型上的成员索引，用于提高过滤效率，作用范围为模型的对象（不管对象是索引添加前创建的，还是索引添加后创建的），不作用于记录，作用是在查询对象时，指定查询索引成员，可以提高查询效率。
索引字段只能是成员，不能是属性，成员类型必须是支持过滤的类型。
运行模型成员索引的作用范围，是运行库中指定GUID运行模型所有版本的所有对象。
运行库中的运行模型，初始没有成员索引。
支持类型：业务物、组织、岗位、人员、业务事、单灵敏时序物、多灵敏时序物、时序事模型。

 对象的成员普通索引，功能与运行模型成员索引功能相同，也叫过滤索引，用于提高查询过滤效率。在历史计划库，对象成员普通索引的作用范围是对象的记录（不管记录是索引添加前创建的，还是索引添加后创建的），可以提高查询记录的效率。

 普通索引字段只能是成员，不能是属性，成员类型必须是支持过滤的类型。

 对象成员索引，不影响其对应的模型和拷贝模型配置。如按同一模型拷贝A创建的对象1、对象2，其中对象1创建了成员1为成员索引，则对象2的成员1不是成员索引，除非为对象2创建成员1为成员索引。

 支持类型：业务物、组织、岗位、人员、业务事对象。

 例如： 对象1有10万条记录，如果给对象的成员m1加索引，再按m1过滤查找记录时，可以提高过滤性能。

# 对象的成员唯一索引

唯一索引表示字段的值不能重复存在，比如工序编号不能重复存在。唯一索引可以指定多个字段，表示多个字段值不能完全相同。
对象上加唯一索引，作用范围为记录，约束的是对象的记录数据，表示多个索引字段的组合值在该对象的所有记录中是唯一的，不能完全相同。如果要添加的唯一索引字段在作用范围内已经存在重复的组合值，则添加唯一索引失败。
可以存在多条记录的唯一索引字段值都为null，null和null不算重复。
唯一索引可以提高过滤效率。唯一索引的过滤效率与索引的顺序有关，遵循最左前缀匹配原则。唯一索引的字段顺序不影响“组合值唯一”的约束逻辑，但是影响索引能否被过滤利用，需将高频查询字段和过滤性强的字段放在左侧。
比如唯一索引的顺序是 “a,b,c”,那么：
能命中索引的过滤条件： a==? 、a==?&&b==?、a==?&&b==?&&c==?
无法命中索引的过滤条件：b==?、b==?&&c==？（由于缺少最左侧字段a）
建议用户将高频过滤的字段放前面，比如经常单独过滤a，则a应该放在索引的左侧；将过滤性强的字段放前面（比如“学号”比“性别”过滤性强），过滤性强的列放前面，能更快缩小查询范围，减少索引扫描量。
历史计划库目前支持添加唯一索引的对象类型：业务事对象。
唯一索引字段都只能是成员，不能是属性。
唯一索引字段支持类型：字符串型、日期时间类型、整型、decimal、浮点型。
注：float和double可以用于唯一索引，但是由于浮点数在计算机中是以二进制存储的，存在精度误差，即使使用默认精度（有效数字位数）来判断数据的唯一性，也可能会出现随机误差，一些在十进制下看似不同的数，在转为二进制存储后可能因为精度问题而变得相同，从而导致唯一索引的约束存在问题，因此，将float和double类型用于唯一索引时仍需谨慎。

# 对象的成员外键

外键用于当前库多个对象记录间关联关系的机制，用于保证数据的完整性和一致性。建立外键字段，只是对象成员字段与另一个对象的成员字段建立引用关系。一个对象可以添加多个外键字段。
使用外键功能，会降低性能。
时空库提供单库外键，不支持跨库外键功能。
支持类型：业务事对象，支持业务事对象的记录之间的外键功能。
例如：对象1的成员A设为外键，该外键关联了对象2的成员B，对象1称为从对象，其成员A称为外键，对象2称为主对象，其成员B称为主键。
外键对添加和修改从对象的记录有约束作用：从对象添加/修改记录时，如果添加/修改记录的外键成员的值，在主对象中不存在主键为此值的记录，则添加/修改记录失败。
外键对删除和修改主对象的记录也有约束作用，采用的是严格模式，主对象删除/修改记录时，若从对象下存在引用记录，则不能删除主对象的该记录，不能修改主对象的该记录的主键值（可以修改其它成员值）。
给对象添加外键时的要求：
1、外键与主键的成员类型必须严格一致，否则添加外键失败。
2、需要保证一个对象只能有一个主键，否则添加外键失败。
3、已经存在记录的对象的成员被设为外键时，必须保证外键的值与主键的值对应起来，否则添加外键失败。
4、已经存在记录的对象的成员被关联为主键时，必须保证主键值不能为空，不能重复，否则添加外键失败。注意：外键添加成功后，仍需要持续保证主键的值不能为空，不能重复。

# 逻辑计算

除了统计计算，还有逻辑判断计算的情况，逻辑计算表达式的参数结构：
fieldExpression
{
 stringMap [] exp;//逻辑表达式[]
 string realValue; //条件为真时的字段名称
 string fadeValue; //条件为假时的字段名称
};

exp
{
 int32 operator; //逻辑运算符
 int32 linkOpera; //条件字段连接条件
 string caseName; //条件字段
 string logicValue; //逻辑值
};
exp
逻辑表达式[]，最多64个。多个条件字段进行连接时，按从前到后的顺序执行。
realValue
exp条件为真时的字段名称，可省，如果省略了，条件为真时返回空。realValue和fadeValue不能同时省略。
fadeValue
exp条件为假时的字段名称，可省，如果省略了，条件为假时返回空。

operator
逻辑运算符，1：等于；2：大于；3：小于；4：不等于；5：大于等于；6：小于等于；7：包含（字符串），8：不包含。

包含与不包含，支持类型：字符串、枚举、引用，其中枚举类型按枚举项名称（itermName）、引用类型按名称（name）进行处理，表示caseName条件字段中，包含/不包含logicValue值。
linkOpera
条件字段连接条件，与下一个条件字段进行连接；=1：与；=2：或，默认为与。
caseName
条件字段，字段值支持数值、字符串、datetime、decimal、枚举类型、引用类型。枚举类型按枚举项名称（itemName）进行处理，引用类型按名称（name）进行处理。
logicValue
逻辑值，字段值类型需要caseName类型相同。

## 审计

 审计，是指主体对客体操作行为进行记录，用于事后追溯。这里操作的客体是对象，一般对关键对象的操作建议配置审计策略。如设备的启停记录，设备运行如果出现异常，会追溯设备的启停历史，分析设备运行时长等信息来进行排查问题。

 各子系统对象的审计策略配置数据的读写功能（审计策略对象的管理），和各子系统对象的审计日志数据的读写功能，都由审计库提供。

 时空库不提供对象审计策略配置和审计信息查询，只是根据审计规则产生审计信息，送到审计库生成审计日志，用于事后追溯。

# 事务

时空对象、数据对象和记录的创建、删除、修改、查询、订阅支持事务，保证操作的原子性、一致性。
时序对象的创建、删除支持事务，时序对象的其他操作比如修改、查询不支持事务。
记录的事务是按记录行加锁，修改对象记录的操作如果加了事务，只有修改的那几条记录加了事务锁，不影响其它记录的修改操作。

## 事务

# 入参校验规则

函数入参如果传少了，会报错，但是传多了，不会报错。例如：对于没有子的对象、记录类型，deleteMode相当于是多余的、无效参数，不管传不传，传的是什么，都是无效的，不报错。 对于有子的类型，deleteMode没传，就按默认的1处理，如果传了，就只能传1,2 ，传其它的值，就报错。

### 入参校验规则

 函数入参如果传少了，会报错，但是传多了，不会报错。例如：对于没有子的对象、记录类型，deleteMode相当于是多余的、无效参数，不管传不传，传的是什么，都是无效的，不报错。 对于有子的类型，deleteMode没传，就按默认的1处理，如果传了，就只能传1,2 ，传其它的值，就报错。

# 大小写规则

函数入参字段名均不区分大小写，包括属性名称、成员名称、参数名称，不区分大小写。
函数输出字段区分大小写，输出一般和输入保持一致，例如：查询返回的数据集列名应和输入的returnFields的大小写一致。如果是输出的是系统字段，如output的参数，其大小写需和产品定义保持一致。例如：查询成员列表属性$memberList、$sensitiveMemberList，返回的成员的name属性字符串值是和成员名称原本的大小写一致。

### 大小写规则

 函数入参字段名均不区分大小写，包括属性名称、成员名称、参数名称，不区分大小写。

 函数输出字段区分大小写，输出一般和输入保持一致，例如：查询返回的数据集列名应和输入的returnFields的大小写一致。如果是输出的是系统字段，如output的参数，其大小写需和产品定义保持一致。例如：查询成员列表属性$memberList、$sensitiveMemberList，返回的成员的name属性字符串值是和成员名称原本的大小写一致。

# returnFields校验规则

如果returnFields里的字段如果都不存在要报错，如果是有的对象存在，有的对象不存在，则不报错，不存在的对象对应字段填空值，对齐。
例如，按模型、拷贝查询，如果returnFields里有模型、拷贝没有的字段，要报错。
例如，查多个对象，如果returnFields里有所有对象都不存在的字段，要报错，如果是有不是所有对象都公共的字段，则不报错，非公共字段，不存在对应字段的对象填空值对齐。
如果returnFields里含有重复字段，返回数据集的列名，从重复的列开始自动加后缀。
例如，returnFields={"mem1","mem1","mem1","mem1"}，则返回DataSet的列名为："mem1","mem1_1","mem1_2","mem1_3"。

# 空值null

不支持给字段赋空值null，空值，表示的是字段未赋值且没有默认值的情况。唯一索引的成员值可以为null，null和null不算重复。

0:{stringValue:"龙卷风"}
空值的情况，查询返回数据为：
0:{NULL:null}
1、分组字段：
null作为一组，作为第一组。
2、聚合字段：
默认宽松模式，忽略null值数据。
严格模式，统计null值数据；对于count外的其他计算，null参与计算，意味着结果返回null。
3、去重：
null值算重复，例如,returnFields={“m1”,”m2”}时，有以下数据：
//m1,m2
24,null
35,null
24,null
以上第1、3行算重复，只保留其中一行即可，去重后：
24,null
35,null
4、排序：
null值当做最小值处理。

# DataMerge（插入记录策略）

1
INSERT
如果n 
 
 
 
 dataMerge 
 插入记录策略，在对单灵敏、多灵敏时序物对象的缺省记录版本的t1时刻上添加记录，如果缺省记录版本的t1时刻不存在记录，则直接添加，如果在t1时刻已存在n条记录，有以下几种处理方式，默认=1。目前只支持方式1。 
 
 
 
 1 
 INSERT 
 如果n<8，并且n+t1时刻新插入条数<=8，则插入，否则失败。 
 
 
 2 
 FAIL 
 不插入，返回失败，会导致同一个事务里的操作回滚。 
 
 
 3 
 IGNORE 
 不插入，忽略，不返回失败，不影响事务里其它操作。 
 
 
 4 
 REPLACE 
 用新记录替换t1时刻已存在的最新版本（第n条）记录，保持t1时刻的记录条数不变。 
 
 
 5 
 REMOVE 
 删除t1时刻已存在的n条记录，添加新记录，添加后的记录的version=0。
