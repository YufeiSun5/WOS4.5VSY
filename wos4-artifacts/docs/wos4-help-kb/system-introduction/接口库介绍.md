# 1. 接口库概述

## 1. 接口库概述

# 1.1 基本概念

接口库是元语言平台的核心通信模块，用于实现APP对象与外部第三方系统之间的数据通信。接口库提供了一套完整的通信机制，支持多种网络协议和通信方式。

### 1.1 基本概念

 接口库是元语言平台的核心通信模块，用于实现APP对象与外部第三方系统之间的数据通信。接口库提供了一套完整的通信机制，支持多种网络协议和通信方式。

# 1.1.1 接口变量

APP接口变量是APP对象的脚本变量，是APP对象和外部第三方系统进行通讯的变量。通过脚本操作APP接口变量，可以控制APP引擎调用接口库IO模块与第三方系统进行通讯。
接口库IO模块是功能单元库的通讯模块，负责与第三方服务或者客户端建立通讯链路，处理FU和外部系统之间的数据通信。

#### 1.1.1 接口变量

# 1.1.2 接口性质

通讯接口有2类性质：服务类接口和客户端类接口，APP接口变量也分3类性质：
**服务类接口变量**：对外提供服务的接口变量，被动等待客户端连接
**客户端类接口变量**：主动连接服务的接口变量，向服务发送请求并等待响应
**移动客户端类接口变量**：移动设备专用的接口变量

**全局作用域**：在APP整个生命周期内有效，包括APP对象、功能函数、块
**APP局部变量**：仅限于定义它的函数内部

## 3. 属性介绍

# 3.1 接口变量属性概述

接口变量有三类属性：基本属性、特有属性。

### 3.1 接口变量属性概述

 接口变量有三类属性：基本属性、特有属性。

# 3.1.1 基本属性

基本属性是接口变量的通用属性，包括静态基本属性和动态基本属性：
**静态基本属性**：如name（名称）、class（接口类型），在创建时指定，其中class为只读属性，创建后不能修改
**动态基本属性**：如status（通信链路状态）、maxRecvCapacity（最大接收容量），反映接口变量的运行状态
**状态属性说明**：
服务类接口变量的Open()和Close()方法控制服务的开关
客户端类接口变量的Open()和Close()方法控制连接链路的开关
status属性表示服务或连接的状态，由系统自动赋值，为只读属性
**最大接收容量**：指的是接收缓存通道的最大容量，单位为字节。当超过设置的最大容量时，系统会释放最旧的数据，并在系统日志中记录缓存区溢出日志。

# 3.1.2 特有属性

特有属性是与具体接口类型相关的通信参数：
**服务类接口变量**：记录对外提供服务的通信参数，如服务IP、监听端口号等
**客户端类接口变量**：记录连接访问服务的通信参数，如连接服务器IP、访问端口、连接字符串等
**协议相关属性**：
不同的业务场景使用不同的通信协议，接口库IO模块支持多种标准通信协议，包括TCP、UDP、MQTT、HTTP、OPC、Kafka、SQL等。不同协议的接口变量具有不同的通信参数配置。

**关闭**：停止通信，断开连接或关闭服务
**删除**：删除接口变量，删除前会自动取消订阅、断开连接或关闭服务
