# 1.操作系统审计

操作系统审计即对Linux系统的操作进行审计，如删除系统文件、关闭防火墙等行为。操作系统审计由用户配置，没有默认审计的行为；成功创建审计策略后，需要将该审计策略关联给某一节点，之后该审计策略生效。由于系统日志无法实时获取，当前方案为每10s获取一次并记录到Wellinos的审计系统中。
操作系统审计有三个子模块有：审计策略管理、节点关联审计策略、操作系统审计记录；

## 1.1.审计策略管理

# 1.1.1.审计策略列表

1.搜索：按ID、名称、客体类型模糊搜索；
2.列表中展示的字段信息：ID、名称、操作系统客体类型、客体信息、操作类型、附加生效条件、操作（编辑、删除）；
![](media/OSystem_3.png)
客体类型，即系统对某客体进行操作。包括文件系统和系统调用；
客体信息，文件系统对应的就是某个文件或者文件夹全路径；系统调用为系统调用名称。
附加生效条件，从任务、用户、退出方式，进一步指定审计策略的生效条件，支持多个条件。

# 1.1.2.新建

1.点击【新建】按钮，输入名称、选择客体类型/操作类型、输入客体信息/附加生效条件即可创建成功。
![](media/OSystem_4.png)

### 1.1.2.新建

 1.点击【新建】按钮，输入名称、选择客体类型/操作类型、输入客体信息/附加生效条件即可创建成功。

 

# 1.1.2.编辑

1.点击【编辑】按钮，打开编辑面板，操作同”新建“。
![](media/OSystem_6.png)

### 1.1.2.编辑

 1.点击【编辑】按钮，打开编辑面板，操作同”新建“。

 

# 1.1.3.删除

1.选中目标行，点击【删除】按钮，打开二次确认面板进行再确认（注：被关联的审计策略无法被删除）。
![](media/OSystem_5.png)

### 1.1.3.删除

 1.选中目标行，点击【删除】按钮，打开二次确认面板进行再确认（注：被关联的审计策略无法被删除）。

## 1.2.节点关联审计策略

# 1.2.1.节点关联审计策略列表

1.左侧结构树：枚举仓库结构树；
2.右侧列表中展示再结构树中所选的仓库所关联的审计策略信息；
![](media/OSystem_7.png)
3.点击【关联审计策略】即可打开配置面板；
![](media/OSystem_8.png)
4.面板中客体类型栏枚举所创建的审计策略；右侧为添加需要关联的审计策略（支持多个添加）；

### 1.2.1.节点关联审计策略列表

 1.左侧结构树：枚举仓库结构树；
