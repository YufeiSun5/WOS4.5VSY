# 备份恢复管理

# 1.备份

备份提供两种方案：手动备份和自动备份。手动备份为实时备份，配置完备份策略后手动开启备份，仅执行一次；自动备份可给不同的仓库配置不同的定时备份策略，定时备份可按照自动备份周期、首次备份时间等策略进行备份。

## 1.备份

 备份提供两种方案：手动备份和自动备份。手动备份为实时备份，配置完备份策略后手动开启备份，仅执行一次；自动备份可给不同的仓库配置不同的定时备份策略，定时备份可按照自动备份周期、首次备份时间等策略进行备份。

# 2.1手动备份

手动备份之前，用户需要先进行备份策略配置：
 对象选择 ： 云 / 端 ；必填项，默认“云”。“云”指（接入到云中心的）节点列表；端指（接入到云中心）的固定端列表，一般为盒子或生产现场的小型设备，也称“微云”
 仓库选择 ：必填项，备份以仓库为粒度，默认 “初始仓库” 
 数据区选择 ：必填项，当备份对象为云时，需要选择指定仓库下的数据区，默认 “所有数据区” ，可以多选。当备份对象为端时，该选项为禁用状态
 端区域选择 ：必填项，当备份对象为端时，需要选择指定仓库下的固定端，默认为 “所有固定端列表” ，可以多选。当备份对象为云时，该选项为禁用状态
 节点IP ：备份位置，必填项，可以是物理机器或虚拟机IP
 备份路径 ：备份机器对应的路径；必填项，例如/usr/backup
 连接用户 ：所备份机器的访问用户名；必填项
 密码 ：所备份机器的访问密码；必填项
策略配置完成以后，点击 【开始备份】 按钮，打开备份详情弹窗。

### 2.2自动备份

 不同仓库数据可以有不同的备份策略， 自动备份策略创建完立即会触发备份执行 。

 

# 2.2.1添加自动备份策略

点击 【创建策略】 按钮，打开弹窗。输入完以下参数即可完成策略的新建。
 备份类型 ： 全量 / 增量 ；必填项，默认“全量”。全量备份是将所有数据进行一个完整的备份；增量备份是在全量备份或上一次增量备份的基础上，只备份与前一次相比变化的数据。

 仓库选择 ：必填项，备份以仓库为粒度，默认 “初始仓库” 

 数据区选择 ：必填项，当备份对象为云时，需要选择指定仓库下的数据区，默认 “所有数据区” ，可以多选。当备份对象为端时，该选项为禁用状态

 节点IP ：备份位置，必填项，可以是物理机器或虚拟机IP

 备份路径 ：备份机器对应的路径，例如/usr/backup

 连接用户 ：所备份机器的访问用户名；必填项

 密码 ：所备份机器的访问密码；必填项 

 自动备份周期 ：备份周期；必填项 ，默认 24 h 

 首次备份时间 ：必填项，默认 当前时刻延后2 h 

 存储时长 ：默认0 单位天

# 2.2.2删除自动备份策略

当用户想取消自动备份策略，可以通过点击【删除】按钮进行自动备份策略的删除操作，删除之后，则会取消当前时间之后的自动备份。

# 2.恢复

手动恢复之前，用户需要先进行恢复策略配置：
 对象选择 ： 云 / 端 ；必填项，默认“云”。“云”指（接入到云中心的）节点列表；端指（接入到云中心）的固定端列表，一般为盒子或生产现场的小型设备，也称“微云”
 仓库选择 ：必填项，恢复以仓库为粒度，默认 “初始仓库”

 恢复路径 ：恢复机器IP对应的路径；必填项，例如/usr/backup/recovery20240512
 连接用户 ：连接恢复机器的用户名；必填项
 密码 ：连接恢复机器的访问密码；必填项
策略配置完成以后，点击 【开始恢复】 按钮，打开恢复详情弹窗。

# 3.查看备份恢复记录

系统会记录备份和恢复操作，便于用户追溯。
当操作对象为云时，除了可以查看整体的备份记录，还可以以数据区为粒度查看操作记录。备份恢复记录可以由用户根据时间范围去删除。
