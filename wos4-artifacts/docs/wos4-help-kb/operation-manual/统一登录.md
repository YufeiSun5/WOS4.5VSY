 WellinOS系统是三权分立的系统，存在4套用户，分别为业务、运维、安全、审计，在整个操作系统的使用过程中，有些操作必须登录相关身份账号才可以进行操作，为了避免每一个操作中有进行登录，所以将登录进行统一管理。如下图所示：

# 1.1. 添加云

BS模式只有当前云，无法再增加云账号。CS模式可以进行添加云，管理端口默认是业务端口+2。
![](media/添加云.png)
添加云时，可以同时进行业务、安全、运维、审计的账号登录，用户可以自选，默认只有业务。
![](media/类型选择.png)

## 1.1. 添加云

 BS模式只有当前云，无法再增加云账号。CS模式可以进行添加云，管理端口默认是业务端口+2。

 

 添加云时，可以同时进行业务、安全、运维、审计的账号登录，用户可以自选，默认只有业务。

 

# 1.2. 编辑云信息

每一个云分为基础信息和账号信息两部分：
**编辑本端：**基本信息都不可编辑，其他内存均可编辑；
![](media/编辑本端.png)
**编辑当前云：**基础信息的名称、云ID、业务ID、业务端口是不可编辑的，并且业务账号无法退出登录，其他内容均可编辑；
![](media/编辑本云.png)
**编辑添加云：**基础信息的云ID是不可编辑的，其他内容均可编辑，当修改了IP和端口后，对应账号会自动进行重新登录。
![](media/编辑其他云.png)
账号信息中可以分别就业务、运维、审计、安全进行分别登录，当选中行未非业务行时，可以有复制按钮，点击复制按钮自动将业务账号信息复制到该行，满足快捷登录；

## 1.2. 编辑云信息

 每一个云分为基础信息和账号信息两部分：

 编辑本端： 基本信息都不可编辑，其他内存均可编辑；

 

 编辑当前云： 基础信息的名称、云ID、业务ID、业务端口是不可编辑的，并且业务账号无法退出登录，其他内容均可编辑；

 

 编辑添加云： 基础信息的云ID是不可编辑的，其他内容均可编辑，当修改了IP和端口后，对应账号会自动进行重新登录。

 

 账号信息中可以分别就业务、运维、审计、安全进行分别登录，当选中行未非业务行时，可以有复制按钮，点击复制按钮自动将业务账号信息复制到该行，满足快捷登录；
