# 日志

 日志功能包括 系统时间记录 、 黑匣子记录 、 追踪日志 以及其他日志，其他日志包含 程序日志 和 系统日志 。

# 1.系统事件记录

系统事件记录是各个子系统产生的一些重要事件，例如 数据区安装/卸载 、 进程启停 、 节点安装/卸载 等。
系统事件记录包含以下信息供用户查看：
事件发生时间
/
节点ID
事件发生对应的机器信息
服务类型
实时库 / 历史库 / 计划库等
日志等级
故障 / 警告 / 事件 / 调试 / 描述
系统事件类型
各个子系统对应系统事件类型不一。例如：数据区安装 / 卸载、进程启停、节点安装 / 卸载等
系统事件对象ID
事件发生对应的（操作）对象ID
系统事件操作错误码
/
系统事件描述
具体的描述信息
系统事件记录可根据 时间范围 、 仓库ID 、 节点ID 以及其他过滤条件来配合查询。其他过滤条件包含： 服务类型 、 日志等级 、 事件类型 以及 系统事件对象ID 。
系统事件记录可以由用户配置存储上限，默认300万条，上限2000万条。当数据超出存储上限时，采用先进先出的规则进行清除。

# 2.黑匣子记录

黑匣子记录以文件形式存储，指开发日志中的业务（操作）信息，包含正常和异常记录。
黑匣子文件中的信息包含以下内容：
操作时间
/
目标对象ID
目标对象的ID或者数据记录的ID
操作类型
增加 / 删除 / 修改
黑匣子记录描述
一般为新值 + 旧值
黑匣子文件列表可根据 时间范围 、 仓库ID 、 节点ID 以及 服务类型 来配合查询。点击操作栏的【查看】按钮，将其导出到本地。
黑匣子日志可以由用户配置存储上限，默认1000万条，上限2000万条。当数据超出存储上限时，采用先进先出的规则进行清除。

 黑匣子记录以文件形式存储，指开发日志中的业务（操作）信息，包含正常和异常记录。

 黑匣子文件中的信息包含以下内容：

 
 
 
 参数名称 
 描述 
 
 
 
 操作时间 
 / 
 
 
 目标对象ID 
 目标对象的ID或者数据记录的ID 
 
 
 操作类型 
 增加 / 删除 / 修改 
 
 
 黑匣子记录描述 
 一般为新值 + 旧值 
 
 
 黑匣子文件列表可根据 时间范围 、 仓库ID 、 节点ID 以及 服务类型 来配合查询。点击操作栏的【查看】按钮，将其导出到本地。

 黑匣子日志可以由用户配置存储上限，默认1000万条，上限2000万条。当数据超出存储上限时，采用先进先出的规则进行清除。

 

# 3.追踪日志

以下场景会产生追踪日志： 
 第一种：端 - 服务的读写数据流。例如： 在业务客户端中查询功能单元模型列表 、 在安全客户端中查询用户列表 
 第二种：服务 - 端的通知业务数据流。
 第三种：后台功能单元 - 服务(节点间)的读写数据流。例如： 不同节点：功能单元库与某一子系统的数据交互 
 第四种：后台功能单元 - 服务(节点内)的读写数据流。例如： 同一节点：功能单元库与某一子系统的数据交互 
 第五种：服务 - 后台功能单元(节点间)的通知业务数据流。
 第六种：服务 - 后台功能单元(节点内)的通知业务数据流。
 第七种：服务 - 服务(节点间)的系统业务数据流。例如： 不同节点：子系统之间的数据交互 
 第八种：服务 - 服务(节点内)的系统业务数据流。例如： 同一节点：子系统之间的数据交互 
 第九种：服务 - 传输网关的系统业务数据流。例如： 数据的传输 
 第十种：传输网关 - 服务的系统业务数据流。
 第十一种：管理中心 - 节点进程的管理数据流。例如： 在运维管理中心对进程的启停操作 
 第十二种：管理中心 - 服务实例 - 应用模块的管理数据流。

/
源节点
源节点ID
源服务
源节点服务
目的节点
目的节点ID
目的服务
目的服务
时间
请求或接收的具体时间
用户会话
发起者的用户会话ID
请求命令码
/
收发类型
发送-请求-同步 / 发送-请求-异步 / 接收-请求-同步 / 接收-请求-异步 / 发送-应答-同步 / 发送-应答-异步 / 接收-应答-同步 / 接收-应答-异步
应答状态
正常 / 应答超时 / 应答失败，单个环节信息
系统事件记录可根据 时间范围 以及其他过滤条件来配合查询。其他过滤条件包含： 跟踪码 、 请求码 、 数据类型 、 sessionID 以及 自定义 。
追踪日志可以由用户配置存储上限，默认1000万条，上限2000万条。当数据超出存储上限时，采用先进先出的规则进行清除。
