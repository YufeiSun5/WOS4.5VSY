# 1.报警

系统主要有以下场景发生时会产生报警：
1、节点运行资源达到报警上限；
2、服务运行资源达到报警上限；
3、数据区运行资源达到报警上限；
4、授权临近到期；
5、...
点击顶部右上方的 报警图标按钮 ，打开报警记录查询弹窗，报警记录包含以下内容：
报警时间
触发报警的时间
节点ID
当查询的是对象是云上的报警时，对应的机器信息
区域ID
当查询的是对象是端上的报警时，对应的端云信息
节点进程类型
运维代理 / 系统网络 / 业务网络 / 运维 / 采集器 / 安装工具 / 数据传输 / 安全管理中心 / 审计管理中心 / 审计日志
服务类型
实时库 / 历史库 / 计划库 / 模型库等
报警类型
节点状态 / 节点CPU / 节点磁盘 / 节点内存 / 服务实例状态 / 服务实例CPU / 服务实例CPU / 授权 / 日志等
报警事件
报警触发 / 报警恢复
报警描述
报警信息描述
报警记录可根据 时间范围 、 仓库ID 、 节点ID / 区域ID 、 服务类型 以及 报警类型 配合查询。
历史报警数据可以由用户配置存储上限，默认300万条，上限2000万条。当数据超出存储上限时，采用先进先出的规则进行清除。

# 2.推送配置

（实时）报警记录推送、备份结果推送、授权临近到期均可以按照提前配置的策略进行推送。
点击顶部右上方的 推送配置按钮 ，打开推送配置弹窗，推送方式支持邮箱、企业微信、手机短信，因不同对象触发的报警，推送的对象或者方式并不一样，故推送信息需要按照仓库来配置。
以下是具体的推送配置内容：
**|** 基础配置：
 邮件推送开关 ： 默认开启 ；必填项
 微信推送开关 ： 默认关闭 ；必填项
 短信推送开关 ： 默认关闭 ；必填项
**|** 邮箱配置：
 发件服务器地址 ： 发送邮箱服务器地址IP ；必填项
 发件人地址 ：支持 网易邮箱、QQ邮箱等 ；必填项
 发件人密码 ：发件人邮箱密码；必填项
 收件人地址 ：收件人邮箱地址；可以配置多个，采用 英文逗号 隔开。必填项
