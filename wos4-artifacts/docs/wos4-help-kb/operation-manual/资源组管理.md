# 2.数据区相关功能介绍

系统服务内划分数据区，数据区是数据容器，数据存储在数据区内，数据区能够进一步绑定服务进程绑定的资源，每个数据区能够指定最多3个副本数，副本分布在不同的计算机节点。
数据区副本支持负载均衡，主副本支持写入，所有从副本支持读数据。每个服务提供了默认数据区，方便用户直接使用。数据区相关功能包含新建&安装、编辑、删除&卸载、迁移等。用户切换不同的服务时，可以查看不同的数据区内容。
以下服务不允许对数据区进行相应操作：
管理资源组下的所有服务：既不可安装新数据区，也不可卸载已有数据区；
时空库资源组下的【全局事务管理器】：既不可安装新数据区，不可卸载已有数据区；
网关资源组下的【管理接入网关】和【管理传输网关】：不可卸载默认数据区。
![](media/service006.png)

新建数据区时用户可以配置基本信息、资源上限、报警阈值以及安装信息。
 网关资源组 下数据区可 可配置虚拟IP 。单副本模式下，设置虚拟IP无意义，通过实际的机器IP即可确定对应的网关访问地址；在 多副本模式 下，由于用户层面是不知道实际运行的副本是哪个节点，因此需要配置虚拟IP。

# 2.4数据区迁移

数据区副本支持数据迁移，当节点负载高时可以把数据区副本手动迁移到负载低的节点上。当数据区存在多副本时，只能迁移从副本。

# 3.保险箱相关功能介绍

保险箱应用主要场景：两个OS之间无法通信的情况下的数据迁移。
用户使用保险箱进行数据迁移的流程如下图所示，基本步骤为：
源区域与目的区域分别建立保险箱，需要确保源于目的两个保险箱的密码相同(最终无该要求)，以便保险箱内容可以正常被目的解密。（前置准备）
源区域数据写入源区域保险箱（导出过程）
源区域保险箱将数据进行备份（迁移过程）
目的区域保险箱将源区域保险箱备份的数据恢复（迁移过程）
目的区域将保险箱中的数据读取上来，写入数据库（导入过程）
![](media/service-新增保险箱数据区2.png)
