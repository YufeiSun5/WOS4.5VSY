# 1.报警记录

对于某些重要或影响程度大操作，不仅要进行审计，还要及时采取一定方式通知管理员等相关用户。
根据对象类型进行报警推送，推送方式优先支持邮件推送，根据固定的邮件内容格式向所配置的用户邮箱进行推送。
当前仅支持对某一类型，添加了审计策略的对象进行审计报警。如，为某一业务物对象添加修改的审计策略，同时为该业务物对象类型配置报警规则，之后对该对象的修改操作会在生成审计记录的同时，产生报警信息，根据操作的严重等级确认是否报警。
报警记录模块内包含三个子模块有：”报警规则配置、关联报警策略、报警策略“；

# 1.报警记录

 对于某些重要或影响程度大操作，不仅要进行审计，还要及时采取一定方式通知管理员等相关用户。

 根据对象类型进行报警推送，推送方式优先支持邮件推送，根据固定的邮件内容格式向所配置的用户邮箱进行推送。

 当前仅支持对某一类型，添加了审计策略的对象进行审计报警。如，为某一业务物对象添加修改的审计策略，同时为该业务物对象类型配置报警规则，之后对该对象的修改操作会在生成审计记录的同时，产生报警信息，根据操作的严重等级确认是否报警。

 报警记录模块内包含三个子模块有：”报警规则配置、关联报警策略、报警策略“；

 

# 1.1.报警规则配置

## 1.1.报警规则配置

# 1.1.1.报警规则配置列表

1.搜索：按名称模糊搜索；
2.列表中展示的字段信息：名称、操作类型、严重等级、操作（编辑、删除）；
![](media/CTPolice_2.png)

### 1.1.1.报警规则配置列表

 1.搜索：按名称模糊搜索；

 2.列表中展示的字段信息：名称、操作类型、严重等级、操作（编辑、删除）；

 

# 1.1.2.新建

1.点击【新建】按钮，输入名称、选择库类型及添加操作类型与严重等级即可创建成功。
2.库类型有：历史库对象审计，计划库对象审计，图关系库对象审计，资源库对象审计，接口库对象审计，视频库对象审计，安全对象审计，审计对象审计，运维对象审计，功能单元对象审计；
![](media/CTPolice_3.png)
2.添加操作类型与严重等级，点击”添加“按钮即可在列表中新增一行，点击”删除“按钮即可删除所选中的行；（注：所选的库类型与列表中所选的操作类型、严重等级进行数据联动）；
![](media/CTPolice_4.png)

### 1.1.2.新建

 1.点击【新建】按钮，输入名称、选择库类型及添加操作类型与严重等级即可创建成功。

 2.库类型有：历史库对象审计，计划库对象审计，图关系库对象审计，资源库对象审计，接口库对象审计，视频库对象审计，安全对象审计，审计对象审计，运维对象审计，功能单元对象审计；

 

 2.添加操作类型与严重等级，点击”添加“按钮即可在列表中新增一行，点击”删除“按钮即可删除所选中的行；（注：所选的库类型与列表中所选的操作类型、严重等级进行数据联动）；

 

# 1.1.2.编辑

1.点击【编辑】按钮，打开编辑面板，操作同”新建“。
![](media/CTPolice_6.png)

### 1.1.2.编辑

 1.点击【编辑】按钮，打开编辑面板，操作同”新建“。

 

# 1.1.3.删除

1.选中目标行，点击【删除】按钮，打开二次确认面板进行再确认（注：被关联的报警规则无法被删除）。
![](media/CTPolice_5.png)

### 1.1.3.删除

 1.选中目标行，点击【删除】按钮，打开二次确认面板进行再确认（注：被关联的报警规则无法被删除）。

## 1.2.关联报警策略

# 1.2.1.关联报警策略列表

1.客体对象类型：默认对”历史库对象，计划库对象，图关系库对象，资源库对象，接口库对象，视频库对象，安全对象，审计对象，运维对象，功能单元对象“进行枚举展示；

2.关联报警策略：选择某库对象后点击操作栏中的”关联报警策略“并点击即可打开配置面板；
3.关联报警策略面板中，再左侧客体对象中枚举并展示已创建的报警规则（支持多选），右侧展示从左侧添加的报警规则；
![](media/CTPolice_8.png)

### 1.2.1.关联报警策略列表

 1.客体对象类型：默认对”历史库对象，计划库对象，图关系库对象，资源库对象，接口库对象，视频库对象，安全对象，审计对象，运维对象，功能单元对象“进行枚举展示；

 

 2.关联报警策略：选择某库对象后点击操作栏中的”关联报警策略“并点击即可打开配置面板；

 3.关联报警策略面板中，再左侧客体对象中枚举并展示已创建的报警规则（支持多选），右侧展示从左侧添加的报警规则；
