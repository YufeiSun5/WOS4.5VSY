# 实时、历史、计划三种时态

万事万物都有三种时态：即：历史、实时、计划，表示世间万物现在、过去、将来的事物状态，以实现其全生命周期管理。如：设备机床从计划采购到投入生产最后到报废，WellinOS中以实时、历史、计划覆盖设备机床的整个生命周期
**历史：**过去，可表示设备机床的报废时间段
**实时：**现在，可表示设备机床的生产时间段
**计划：**将来，可表示设备机床的计划采购时间段

# 多灵敏时序物

用来描述**多个值高频存储、变化**的客观世界物体，其数据需要频繁地写库，如：**采集设备（如定时采集温度、压力、流量等多个值）**。
说明：
多灵敏时序物较单灵敏时序物相比，可在一个模型中同时设置多个灵敏值成员，灵敏值之外的其他属性也可使用非灵敏成员表示。
多灵敏时序物可表示SCADA里的结构变量，一个设备有多个采集值，一个典型应用场景：智能电表有2个标签属性：位置和分组，还有3个采集变量：电流、电压、相位。在此场景中，智能电表即为一个多灵敏时序物模型，位置与分组可设置为非灵敏成员，电流、电压、相位则可设置为灵敏成员。
其孪生的多灵敏时序物对象按照时态分为：实时多灵敏时序物、历史多灵敏时序物，描述现在和过去的状态，无计划时态对象。

# 时空对象

WellinOS按照时空体系数字孪生构建，时空对象往往对应企业的组织架构，例如集团-工厂-车间等层级体系。云系统时空下可以创建根时空，根时空下继续创建子时空，形成一棵**时空对象**树，从而对照孪生企业完整架构层级。
**时空对象**类似一个容器，其中包含业务应用相关的内容（程序+数据），包括功能单元对象、数据对象（物、事、人员、组织等）。时空对象之间的操作有时空约束，可以保证业务模块之间的相对隔离，方便系统的维护。

# 功能单元模型

功能单元模型包括：**元语言模型、webjs模型、AppModule**，功能单元模型是**数据模型**与**脚本逻辑**的**结合体**，在功能单元内不但可以编写脚本也可以组装数据模型、其他功能单元模型，每个功能单元模型都是一个模块，每个模块可自由组装。
开发者在进行业务系统开发时，可将预期业务功能拆解设计为独立的功能单元模块，通过组装功能单元模块，进行业务系统的快速搭建。

# 元语言功能单元模型开发

点击桌面上的**时空功能开发**图标，进入业务开发界面。
新建个人仓库，并在新仓库中点击**新建**按钮，选择**功能单元**，新建**元语言功能单元**，名称为：**FU_M_元语言功能单元**
本部分主要体现功能单元的开发操作步骤，所以功能单元脚本以简单打印为例，在**onCreate**中写入以下脚本：
//int类型数据打印
int32 a = 1;
int32 b = 2;
int32 c = a+b;
Trace(3,"c的值为：",c);
点击**编译**，提示**编译成功**即为脚本没有语法问题可以正常运行

# 元语言功能单元部署运行

点击右侧**提交**按钮，输入提交说明（可自定义，如1.0），点击**确认**按钮进行提交
仓库界面，选中功能单元模型，点击上方的**生成拷贝**按钮。选择提交的版本，点击**生成**按钮，即可在仓库中生成一个功能单元拷贝
点击右侧**自启动APP配置**按钮，选择生成的功能单元拷贝，点击**确定**按钮。
点击上方的**打包**按钮，编写**打包说明**（可自定义编写），点击**开始**按钮，即可完成仓库数据包打包。
点击上方的**部署管理**按钮，进入到时空对象管理界面。点击**新建**按钮，输入**时空对象名称**（可自定义输入），点击**确定**按钮完成时空对象创建。
选中创建的时空对象，点击下方时空仓库管理的**添加**按钮，选择之前打包的仓库数据包，点击**确定**按钮，即可完成仓库包添加。
勾选仓库数据包右上角的**星图标**，选中时空对象与仓库包，依次点击时空对象管理中的**部署**与**启动**按钮，即可完成功能单元的部署运行。
点击桌面左下角的**时空对象管理平台**，选择之前创建的时空对象，点击**全部**按钮，即可看到运行的元语言功能单元。点击上方的**日志**按钮，选择**脚本日志**，即可查看功能单元打印的日志信息。
