# 1.业务审计

业务审计即针对业务数据或对象所进行的审计，需要审计管理员创建审计策略对象、配置审计的主体和审计策略，并关联客体业务对象，关联后指定的主体对象对客体的操作将会被记录。如：创建审计策略对象，配置主体为user1，审计策略为修改，并将该审计策略关联实时业务物对象1，则user1对实时业务物对象1 的修改操作将会被记录。
仓库支持一级仓库，可对仓库进行管理，包含新建、编辑以及删除，与此同时，再操作时也需要先选择对应的仓库。不同仓库之间的数据是互相隔离的，而且不支持相互关联。

# 1.2.业务审计策略管理

业务系统的客体对象操作由用户定义是否进行审计，审计管理员创建审计策略对象，并配置审计操作的主体信息、审计策略（操作行为）、审计结果（操作结果），再将该审计策略对象与客体对象进行关联，随后该客体对象符合审计策略的操作将会记录审计信息。
WOS系统默认开启审计，无需配置。
![](media/BArecord_10.png)

## 1.2.业务审计策略管理

 业务系统的客体对象操作由用户定义是否进行审计，审计管理员创建审计策略对象，并配置审计操作的主体信息、审计策略（操作行为）、审计结果（操作结果），再将该审计策略对象与客体对象进行关联，随后该客体对象符合审计策略的操作将会记录审计信息。

 WOS系统默认开启审计，无需配置。

 

# 1.2.1.新建

点击【新建】按钮，打开审计策略创建面板。
![](media/BArecord_11.png)
需要输入基础信息与主体信息即可创建完成；
主体对象类型有：进程用户，安全库用户，功能单元用户；
主体选择：与主体对象类型匹配，不可重复，不可为空，上限1024；
库类型：模型库对象审计，实时库对象审计，历史库对象审计，计划库对象审计，图关系库对象审计，资源库对象审计，接口库对象审计，视频库对象审计，空间库对象审计，功能单元对象审计；
另外，WOS系统即安全、审计、运维子系统，默认非读操作全审计，无需用户层面配置审计策略，因此也无需配置审计策略与客体的关联关系
操作类型：与所选库类型进行数据联动；
