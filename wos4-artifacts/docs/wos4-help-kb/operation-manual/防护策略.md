# 防护策略

防护策略是指WOS中的进程（用户）对象对于文件的访问策略，文件类型包含文件目录、程序文件、数据文件。
文件创建（或者称作关联）了防护策略，那就意味着只有关联该防护策略的进程才可以访问，但访问会受到防护策略具体的规则限制。默认所有文件没有加防护策略，代表进程对象对任何文件的访问不受限制。
以下场景可以考虑采取防护策略：
不同进程对象均可以对某个文件进行访问，但是访问权限不一样，比如存在两个进程对象，进程对象1只可以对其进行读；而进程对象2可以对其进行读写。

# 1.2编辑

点击操作栏【删除】按钮或者列表上方的【批量删除】按钮进行删除。**需要注意的是，已经被进程对象关联的防护策略不允许删除**。

### 1.3删除

 点击操作栏【删除】按钮或者列表上方的【批量删除】按钮进行删除。 需要注意的是，已经被进程对象关联的防护策略不允许删除 。

## 2.进程对象

 WOS进程对象详细信息如下，可以对进程对象的运行进行启用/禁用、防护策略的关联等。

 

# 2.1关联防护策略

点击操作栏中的【关联防护策略】按钮或者列表上方的【批量关联防护策略】按钮进行删除，打开关联防护策略弹窗，一个进程对象可以关联多个防护策略。一个防护策略可以被多个进程对象关联。
![image-20240531180317509](media/image-20240531180317509.png)

### 2.1关联防护策略

 点击操作栏中的【关联防护策略】按钮或者列表上方的【批量关联防护策略】按钮进行删除，打开关联防护策略弹窗，一个进程对象可以关联多个防护策略。一个防护策略可以被多个进程对象关联。

 

# 2.2启用/禁用（防护策略）

关联了防护策略后，用户可以对其进行启用和禁用。点击操作栏中的【启用/禁用防护策略】按钮或者列表上方的【批量启用/禁用防护策略】按钮，打开操作详情弹窗。
禁用的时间段内，防护策略不生效。
![image-20240531180817339](media/image-20240531180817339.png)

### 2.2启用/禁用（防护策略）

 关联了防护策略后，用户可以对其进行启用和禁用。点击操作栏中的【启用/禁用防护策略】按钮或者列表上方的【批量启用/禁用防护策略】按钮，打开操作详情弹窗。

 禁用的时间段内，防护策略不生效。

 

# 2.3允许操作进程

安全管理中允许对进程的启停操作进行授权，点击【允许/禁止进程对象启停】，打开相应的弹窗，查看具体详情。

### 2.3允许操作进程

 安全管理中允许对进程的启停操作进行授权，点击【允许/禁止进程对象启停】，打开相应的弹窗，查看具体详情。
