# 1.新建&安装节点

点击【新建】按钮，进行节点的新建和安装。节点安装本质是安装节点环境，包括节点运维代理、日志采集器、性能采集器等。
配置信息包含：基本信息、连接信息、报警策略配置、安装信息、网卡信息配置。
![002](media/nodeinstall002.png)
网卡用途配置是考虑到节点资源不够的情况下采用多网卡，其中网卡用途配包含以下6个选项：
业务管理主&从：跟时空库交互、功能单元交互，业务上的通信；

系统管理主&从：系统之间的通信，安全、审计、运维等；

数据同步主&从：数据区之间的数据同步。
注意：
①　网卡信息配置不能出现多个网卡对应同一个用途，例如网卡1对应业务管理-主，网卡2也对应业务管理-主；
②　当网卡信息配置需要包含所有用途；
③　所有网卡的配置用途中备可以没有，主必须要有。
资源足够的情况下，只使用一个网卡也可以。网卡信息配置必须是在节点安装前完成，如果不进行配置的话，默认第一个网卡承担所有工作。当节点配置信息填写完成后，可以点击【网卡信息配置】进行网卡配置，也可不进行配置（使用默认规则）直接点击【安装】按钮进行节点安装。需要注意的是：
①　网卡信息获取过程中，不允许进行安装；
②　网卡信息获取过程中，不允许对节点配置信息进行修改；
③　网卡信息获取完毕后，此时再修改IP，需要用户手动重新获取网卡信息，否则还是默认第一个网卡承担所有工作。

# 6.性能监视

节点接入之后，用户可以查看节点在运行过程中的实时数据和历史数据。
性能监视包含CPU、内存、硬盘以及网卡的运行状况，其中硬盘又包含IOStat、Smart信息、磁盘信息。以下是具体监视的参数：
**（1）CPU：**
CPU使用率；

CPU处于空闲状态的百分比；

IO等待所占用的CPU百分比。
**（2）内存：**
内存总量；
已使用内存；
空闲内存量；
缓存内存量；
可立即分配给进程的内存数量。
**（3）硬盘（IOStat）**
每秒对该设备的读请求被合并次数；
每秒对该设备的写请求被合并次数；
每秒完成的读次数；
每秒完成的写次数；
每秒读数据量；
每秒写数据量；
平均每次IO操作的数据量；
平均等待处理的IO请求队列长度；
平均每次IO请求的等待时间；
平均每次IO请求的处理时间；
IO时间所占用1s的百分比。
**（4）硬盘（Smart信息）**
底层数据读取错误率；
启动/停止计数；
重映射扇区数；
通电时间累计；
主轴起旋重试次数；
磁盘校准重试次数；
磁盘通电次数；
温度；
擦除错误块计数；
当前待映射扇区计数；
ULTRA DMA奇偶校验错误率；
写错误率。
**（5）硬盘（磁盘信息）**
分区大小；
可用磁盘容量。
