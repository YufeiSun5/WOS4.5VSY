# 2.用户管理

用户管理负责当前云端用户的新建、展示、修改以及删除等操作。
用户有两类： 普通用户 和 功能单元用户 ，普通用户又可以分为四类（ 系统管理用户 / 安全管理用户 / 审计管理用户 / 业务用户 ），各类用户各司其职，这里区分类别是确定该用户访问的操作范围（满足 强制性访问规则 ）。
功能单元用户和普通用户有以下区别：
普通用户可以登录，功能单元用户不能登录；
功能单元用户不提供分配角色、关联用户组的能力；
功能单元用户的信息只包含用户名、密码。不像普通用户可以设置登录有效期、密码有效期等。

 系统管理集群 ：运维库中的仓库和数据区，其中数据区包含业务数据区和保险箱数据区。

 业务管理集群-仓库 ：业务管理集群下的仓库，涉及的库有模型库、资源库、空间库以及图关系库。

 业务管理集群-时空 ：业务管理集群下的时空对象。

# 7.修改安全标记

安全标记包括了安全级别和类别，任何对象（主体和客体）均拥有安全标记属性。拥有相应权限的用户可以修改安全标记。
主体和客体都应该确定安全级别，级别默认为1（最低级别），级别是一个范围（1~65536），包含最大级别和最小级别。主体和客体都有属于自己的类别，通过主体对客体的访问类别控制来判断是否属于访问范畴内。
用户、角色和用户组无论是作为主体还是客体，安全标记是同一个属性。以修改用户的安全标记为例：

# 9.权限继承

安全对象可以通过具体的操作，来达到权限继承的目的。
 用户继承： 
用户通过关联角色（或者角色关联用户）可以继承角色的权限；
用户被添加到用户组下，可以继承用户组的权限；
 角色继承： 
角色通过关联其他角色能继承关联角色的权限；
 用户组继承： 
子用户组继承父用户组的权限；
用户组通过关联角色，可以继承角色的权限。
