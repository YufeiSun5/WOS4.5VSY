## 系统分层判断

WellinOS4.5 不是单一页面搭建器，而是一个带桌面壳、对象库、运行引擎、运维安全体系的工业数字孪生平台。帮助文档把能力分为以下几层：

1. 桌面与客户端层：提供 BS 浏览器桌面和 CS 客户端桌面。桌面内置时空功能开发、时空对象管理、系统运维、安全管理、审计管理、商店、数据接入、转发、抽取等工具。
2. 数据建模层：以模型、拷贝、对象、记录为主线，把客观对象映射为数字孪生对象。模型是结构，拷贝是配置载体，对象是运行态/业务态实例，记录依附于可记录对象。
3. 时态数据层：核心库分实时、历史、计划三类。实时表示现在，历史表示过去，计划表示未来；三者覆盖对象生命周期。
4. App 计算层：App 引擎负责 App 对象创建、部署启动、反部署停止、查询、修改、订阅、调试、函数调用、接口调用、定时调用和事务控制。语言/运行类型包含元语言、WebJS、NodeJS，手册还出现 C++/QT App 描述。
5. 运维资源层：云区域、节点、机架、资源组、服务、数据区、网关、备份恢复、升级、准入、报警推送和授权管理构成运维体系。资源组按管理、时空库、功能单元、网关等维度隔离调度资源。
6. 安全与审计层：三权/多权身份体系，包含业务、运维、安全、审计用户；安全客户端管理组织、角色、用户、权限、策略、证书；审计覆盖业务对象和操作系统行为，并可联动报警。
7. 接口开发层：手册按子系统提供数据结构、枚举、错误码、JS 函数、元语言函数。开发时先判断前端 JS 还是后台元语言，再按子系统、数据对象、操作类型和已知条件选择接口。

## 对当前页面设计自动化的影响

- 手册没有直接暴露完整的页面设计器组件章节，目录中未检出成体系的“布局/组件/ECharts/表格/树”说明。页面编辑器自动化仍应以运行时 Vue/comMap 实测为准。
- 手册确认 WebJS App 是浏览器侧运行对象，包含 html/css/js/json/自定义文件，这解释了页面预览常在 iframe 中加载，顶层 #app 为空不能作为失败结论。
- 手册确认接口函数分前端 JS 和后台元语言两类。后续页面按钮联动、变量中转、组件刷新如果只影响页面展示，应优先走前端 WebJS/页面 runtime；涉及时空库对象增删改查时，再查对应 JS 函数或元语言函数。
- 手册确认 BS 模式是当前云单云访问，CS 模式可多云。我们的浏览器自动化应继续以当前 BS 桌面和可见菜单层级为准，不能依赖历史动态业务 URL。
- 手册确认桌面任务栏包含帮助手册入口；本次通过右下角帮助图标打开帮助页的路径应写入后续导航技能。

## 初步结论

WellinOS4.5 的核心抽象是“数字孪生 + 三时态对象库 + App 引擎”。页面设计只是 WebJS/App 运行层的一部分；真正的业务数据和权限边界在模型库、实时库、历史库、计划库、安全库、运维库和接口函数体系里。后续做页面 skill 时，应把“页面布局与组件持久化”作为前端 runtime 技能，把“业务数据读写”作为接口函数/时空库技能分开维护。
