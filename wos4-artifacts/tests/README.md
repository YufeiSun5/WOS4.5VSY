# `wos4-artifacts/tests`

测试目录。

用于保存测试用例、测试数据、自动化验证输入和可复跑的测试脚本。测试目录放“怎么测”和“用什么数据测”，不放最终汇总报告。

## 子目录

- `unit/`：单元测试。验证本地脚本、解析器、配置读取、纯函数、数据转换等不依赖 WOS4 在线环境的逻辑。
- `integration/`：集成测试。验证多个模块或工具协同，例如登录、导航、导出、保存、提交、预览链路。
- `regression/`：回归测试。保存已修复问题和关键业务链路的复测用例，防止后续改动破坏。
- `fixtures/`：测试数据、样例 JSON、样例 ini、mock 响应、预期输出。
- `manual/`：人工测试步骤和检查表，适合无法稳定自动化的 WOS4 前台流程。

## 测试记录要求

每个测试用例应写明：

- 测试目标
- 前置条件
- 操作步骤
- 期望结果
- 实际结果
- 关联截图/快照
- 是否可自动化复跑

测试执行后的汇总结论放到 `wos4-artifacts/reports/`。
