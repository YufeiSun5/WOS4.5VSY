# `wos4-artifacts/tasks`

任务工作包目录。

复杂任务按“一个任务一个文件夹”管理。AI 先读任务文件夹的 `README.md`，再读需求、布局、前端逻辑、后端改动、测试验收和执行日志。

## 什么时候用

- 用户给了图片、设计稿或布局描述。
- 一个任务同时涉及前端布局、前端逻辑、后端改动、发布部署或测试。
- 任务需要分阶段完成。
- 任务需要多人或多个 AI 身份协作。

## 命名

目录名优先中文，带日期：

```text
yyyyMMdd-中文任务名/
```

## 标准结构

从 `_template/` 复制：

```text
README.md
01-需求说明.md
02-设计布局.md
03-前端逻辑.md
04-后端改动.md
05-测试验收.md
执行日志.md
assets/
screenshots/
snapshots/
backups/
```

交互面板只写任务工作包路径和简要状态，具体事件写入任务文件夹。
