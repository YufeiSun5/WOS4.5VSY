# 05 测试验收

## 测试类型

- 单元测试：验证假数据过滤、总分计算、CRUD 入参返回结构。
- 集成测试：验证页面按钮调用前端刷新函数和后端 `Call(...)`。
- 回归测试：验证已有 clean CRUD 和蓝色客户端规则未被破坏。
- 人工测试：验证页面截图、弹窗位置、表格高度、图表高度和可见按钮点击。

## 设计阶段验收

- 任务工作包创建完成。
- 参考图已放入 `assets/`。
- preflight 已通过并保存快照。
- 方案中明确区分：
  - 已验证能力。
  - 待实测能力。
  - 推荐路径。
  - 降级方案。

## 布局阶段验收

- 布局先于组件完成。
- 新建 Palimpsest 项目/客户端，不在旧客户端上直接改。
- 重新读取 `comMap.$Children` 后记录布局快照。
- 左侧菜单列和右侧内容列存在。
- 左右两个页面或降级后的两个区域按不同百分比显示，默认左 18%、右 82%。
- 表格父容器高度不低于 320。
- 图表 canvas 高度不低于 240。
- 同行百分比列宽和 `<= 100`。

## 前端逻辑验收

- 打开页面后假数据表格可见。
- 点击查询后表格、KPI 和图表刷新。
- 点击新增打开弹出页或抽屉。
- 提交新增后表格多一条记录。
- 点击编辑/评分后表单回填正确。
- 提交编辑后综合评分变化。
- 点击删除后记录不可见或状态变为逻辑删除。

## 后端验收

- `QueryRecords` 返回 `errorcode=0` 和 records。
- `CreateRecord` 返回新增 record。
- `UpdateRecord` 返回更新后的 record。
- `DeleteRecord` 返回 deletedId。
- 正式客户端可见按钮点击中后端 `Call` 返回 `ret=0`。
- 若后端暂为假数据函数，报告中必须写明不是生产数据库持久化。

## 正式客户端验收

- 通过蓝色 WebJS 客户端对象打开。
- inner `.page_view.__vue__` 挂载真实页面。
- 左菜单、右内容、表格、图表、弹出页都可见。
- 点击前后截图和快照对比能证明数据变化。
- 不接受只在 raw iframe window 中执行脚本作为最终验收。

## 测试证据

后续实施时应保存：

- 编辑器 before/after JSON。
- 编辑器截图。
- 正式预览/蓝色客户端截图。
- 按钮点击前后 JSON。
- 后端 `ret/errorcodes` 快照。
- 测试报告，放入 `wos4-artifacts/reports/`。
