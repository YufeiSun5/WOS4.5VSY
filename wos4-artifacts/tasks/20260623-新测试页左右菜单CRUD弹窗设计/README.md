## 任务目标

设计一张新的管理类测试页，视觉参考用户提供的人才考核管理页。页面要求：

- 页面只有左右两部分：左侧是垂直菜单页面，右侧是主要内容页面。
- 不设计顶部菜单或顶部栏。
- 本任务按新项目/新客户端重新创建，不在旧工程或旧客户端上继续改。
- 两个页面按不同百分比占位：左侧菜单页默认 18%，右侧内容页默认 82%。
- 右侧包含筛选区、统计卡片、图表、表格。
- 支持假数据的查询、新增、编辑、删除。
- 提供一个弹出页，用于编辑或评分。
- 先完成方案设计，不直接修改 WOS4 线上页面。

## 改动范围

- 前端布局：优先设计客户端双页面并排，并保留单画面左右分栏降级方案；两种方案都只保留左右两部分。

## 关键设计结论

- 用户补充判断：大概率可以让一个客户端左侧显示菜单页面、右侧显示内容页面。
- 用户追加约束：不用顶部菜单，只有左右两部分即可。
- 用户追加约束：需要重新创建项目，两个页面设计不同百分比。
- 推荐实施时优先验证“客户端双页面并排”能力；如果验证不稳定，再降级为“一个 WOS4 页面内的左右分栏”。
- WOS4 已验证可以做 `RContainer -> RRow -> RCol` 左右分栏和表格、折线图、输入框、按钮持久化。
- WOS4 客户端可以挂载多个画面并设置首页，但“同一客户端视口中同时显示两个独立页面对象”当前没有充分证据，不能作为第一版默认方案。
- 弹出页优先按“同页遮罩/居中编辑面板”设计；若 WOS4 弹窗组件或页面弹出机制实施时不可稳定持久化，则降级为右侧抽屉式详情区。
