# 方案设计

## 目标和用户场景

- 目标：摸清 `PalimpsestExcel_82` 页面内 Excel 组件能做什么，并把页面改造成可交互的实验页。
- 使用者：开发人员和后续 AI，用于判断 Excel 组件读写、样式、导出、打印分页能力。
- 业务场景：在同一页面里输入单元格地址和值，点击按钮验证读取、写入、样式、导出、打印等行为。
- 成功标准：每项能力都有运行证据；不支持或未找到入口的能力也有探针记录和边界说明。

## 现有系统核实

- 已确认存在的路径：用户指定 `盛云科技_孙宇飞_Palimpsest_前端_0623 -> Palim -> PalimpsestExcel_82`。
- 已确认存在的组件：页面内有一个 Excel 组件，真实原型和 API 待 live 探针。
- 已确认工具状态：基础 preflight 通过；`slot2` 配置为 `xiang_xuezhi`，当前尚未启动；`slot1` 活跃但不是本轮账号。
- 已确认规则：布局必须先于组件；修改前必须做页面运行态可回灌备份。

## 缺口和风险

- Excel 组件可能不是项目已验证组件，不能直接复用表格/图表持久化结论。
- 导出可能触发浏览器下载权限或弹窗，需要实时观察。
- 打印分页能力可能依赖浏览器打印界面，自动化证据可能只能做到打印/导出前状态和生成文件检查。
- 如果页面当前编辑器已有未保存变更，必须先备份并判断是否继续，不能覆盖用户未保存内容。

## 推荐方案

1. 启动 `wos4_slot2`，申请 `xiang_xuezhi` 账号锁。
2. 按可见菜单进入建模系统客户端和目标页面。
3. 扫描顶层弹窗，确认 `#page_edit_view_area` 和 `comMap.$Children`。
4. 创建 `before-runtime.json`、`restore-runtime.js`、`before-editor.png`。
5. 运行 Excel 组件能力探针，保存方法/属性/DOM/截图证据。
6. 按布局方案重建或增量调整页面结构，先保存布局快照。
7. 放置实验控件并写入按钮脚本。
8. 保存、提交、重开编辑页、fresh 预览验证。
9. 汇总 Excel 能力矩阵和打印分页结论到任务包与 `WOS4_deep_test_notes.md`。

## 不推荐路径

- 不用旧动态 URL 直接打开编辑器。
- 不在未备份前清空或覆盖 Excel 组件。
- 不把运行态临时注入当作持久化完成。
- 不用旧预览 tab 判断最新结果。

## 阶段计划

| 阶段 | 负责人 AI | 输出 | 验收标准 |
|---|---|---|---|
| 页面核实 | 向学智_frontend-ai | before 备份、页面截图 | 指定页面和 Excel 组件确认存在 |
| 能力探针 | 向学智_frontend-ai | capability JSON | 原型、方法、属性、DOM 特征清晰 |
| 布局实施 | 向学智_frontend-ai | rowsManager 快照 | Excel 主区和实验区尺寸稳定 |
| 脚本实验 | 向学智_frontend-ai | 按钮脚本、验证快照 | 读写、样式、导出、打印分别有结论 |
| 验收记录 | 向学智_frontend-ai | 报告和 notes | 提交、重开、fresh 预览通过或明确 blocked |
