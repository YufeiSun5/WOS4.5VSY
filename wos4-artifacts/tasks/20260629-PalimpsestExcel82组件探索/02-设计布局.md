# 02 设计布局

## 布局原则

严格按 `wos4-layout-devtools-skill` 执行：先稳定 `RContainer/RRow/RCol` 布局结构，再向空槽位放控件。布局完成后重新读取 `comMap.$Children` 和 `rowsManager`，确认目标槽位、尺寸策略和空列存在，再进入组件阶段。

## 初始目标布局

最终结构以当前页面探针结果为准，初始建议：

```text
RootLayout
├─ HeaderRow fixed 64
│  └─ TitleAndStatusCol 100%
├─ MainRow remaining
│  ├─ ExcelHostCol 68%
│  └─ ExperimentPanelCol 32%
│     └─ ExperimentPanelContainer
│        ├─ CellAddressRow fixed 64
│        ├─ CellValueRow fixed 96
│        ├─ OperationButtonsRow fixed 160
│        ├─ StyleButtonsRow fixed 120
│        ├─ ExportPrintRow fixed 120
│        └─ ResultLogRow remaining
└─ FooterRow fixed 56
   └─ EvidenceStatusCol 100%
```

## 实验区控件预留

- 单元格地址输入：默认 `B1`。
- 单元格值输入：用于写入文本/数字。
- 操作按钮：读取、写入、清空、填充样例数据。
- 样式按钮：设置背景色、字体、边框、合并/取消合并按探针能力决定。
- 导出/打印按钮：导出 Excel、导出 PDF 或打印预览按组件能力决定。
- 结果日志：展示最后一次操作结果、错误码、返回值摘要。

## 尺寸策略

- 顶部标题区固定高度，避免挤压 Excel 主区。
- 中部主行使用剩余空间，Excel 父容器必须有可见高度。
- 实验区右侧列使用百分比宽度，同行逻辑宽度和不超过 100。
- 右侧实验区内部按钮行固定高度，日志行吃剩余空间。
- 底部状态区固定高度，记录当前提交/预览/导出结果。

## 验收条件

- Excel 组件父容器高度可用，不塌陷。
- 实验区控件不遮挡 Excel 主区。
- 预览首屏可同时看到 Excel 主区和实验区。
- 移动或窄视口下不出现同行列宽总和超过 100 导致的异常换行。
