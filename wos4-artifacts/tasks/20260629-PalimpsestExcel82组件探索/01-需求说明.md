# 01 需求说明

## 用户原始描述

用户指定本轮使用向学智账号，slot2，打开建模客户端，进入：

```text
盛云孙宇飞根组
-> 盛云科技_孙宇飞_Palimpsest_前端_0623
-> Palim
-> PalimpsestExcel_82
```

页面里已有一个 Excel 组件。用户不清楚该组件怎么用，希望 AI 先摸清楚组件能力边界，再在这个页面内重新布局，留出实验操作区，例如输入 `B1` 读取指定单元格、往单元格写东西、画样式、探索导出和打印分页时表头表尾是否能自动分页。

## 业务目标

把 `PalimpsestExcel_82` 从单纯组件展示页改成可复用的 Excel 组件实验页，明确记录：

- 组件实例的原型、关键属性、公开方法和持久化字段。
- 单元格读取、写入、样式设置的可用方式。
- 导出能力和导出文件内容/样式保真边界。
- 打印或导出时表头、表尾、分页能力的真实结果。

## 输入材料

- 图片：无。
- 参考页面：WOS4 页面 `PalimpsestExcel_82`。
- 现有 WOS4 路径：建模系统客户端内 `盛云科技_孙宇飞_Palimpsest_前端_0623`。
- 现有后端/模型：本轮默认不依赖后端。
- 现有数据结构：待从页面运行态 Excel 组件探针读取。
- 用户期望利用的已有数据：页面内已有 Excel 组件。

## 范围

本次要做：

- 使用向学智账号 slot2 打开指定页面。
- 修改前创建可回灌页面运行态备份。
- 探针分析 Excel 组件实例、方法、属性和 DOM。
- 先重新布局，预留 Excel 主区和实验操作区。
- 放置并配置实验控件，脚本证明读、写、样式、导出、打印分页能力。
- 提交、重开编辑页、打开预览页验证。

本次不做：

- 不修改其他 Palimpsest 页面。
- 不新增后端函数。
- 不在未验证前声明 Excel 导出或打印分页一定支持。
- 不复用陈旧预览页作为验收依据。

## 待确认问题

- Excel 组件的真实 toolkit id、prototypeName 和 API 尚待页面探针确认。
- 导出是否会触发浏览器下载、是否需要用户确认下载权限，需运行时观察。
- 打印能力可能只暴露浏览器打印或组件内部打印入口，需实际探针后再定。
