# 前端逻辑

## 假数据结构

```js
{
  code: "EQ-2026-001",
  name: "一号产线视觉检测机",
  area: "总装车间",
  status: "运行中",
  owner: "李明轩",
  health: 96,
  lastMaintain: "2026-06-20",
  parts: [
    { name: "相机模组", status: "正常", value: "99%" },
    { name: "传送机构", status: "关注", value: "82%" }
  ]
}
```

## 前端假查询数据源

- `assets/pal-expand-table-demo-data.json` 同时保存 `treeNodes` 和 `rows`。
- `treeNodes` 只描述树节点和筛选条件，例如 `{ area: "二车间" }`、`{ status: "异常" }`。
- `rows` 保存完整设备假数据。
- 页面运行时通过 `queryRows(node)` 按节点 `filter` 过滤 `rows`，模拟真实接口查询结果。
- 当前不调用后端、不调用时空对象、不调用自定义计算。

## 交互

- 左侧树选择节点后，右侧表格调用前端 `queryRows(node)` 切换数据。
- 当前实现写入 `pal_expand_native_table.detailConfig`，先调用 `this.table(...)` 渲染原生表格，再给第一列单元格安装点击事件。
- 点击行首 `展开` 后，在主表体当前行后插入 `.pal-expand-detail-row`，第一列文本切换为 `收起`；再次点击移除明细行。
- 明细行展示设备编码、区域、责任人、健康度、维护建议和部件状态。
- 操作列保持 `fixed:false`，避免参考页 fixed-right 克隆表体导致展开行错位。
- 状态列在运行时渲染为胶囊标签，健康度列渲染为数字 + 条形进度，操作列渲染为 `查看 / 编辑 / 更多` 按钮。
- `查看` 按钮复用当前行展开逻辑；`编辑 / 更多` 当前只做前端 toast 演示反馈，不接真实后端。
- 搜索和筛选本轮只做静态演示，不接后端。
