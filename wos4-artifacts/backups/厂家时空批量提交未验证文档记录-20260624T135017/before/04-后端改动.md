# Palimpsest 后端数据库与方法设计

## 范围

本设计服务 `盛云科技_孙宇飞_Palimpsest` 新测试页，目标是支撑左侧菜单、右侧实习生考核管理列表、评分弹窗和假数据 CRUD。当前先作为建模系统后台模型设计稿，不直接声明已完成后端对象落地。

## 菜单结构

客户端只保留左右两部分，不使用顶部菜单。左侧菜单建议由后端返回树形数据，前端按 `sort_no` 排序。

| 菜单编码 | 名称 | 页面 | 说明 |
|---|---|---|---|
| `assessment_dashboard` | 考核总览 | `PalimpsestContent_82` | 当前主页面，含筛选、KPI、图表、表格 |
| `student_assessment` | 实习生考核管理 | `PalimpsestContent_82` | 默认选中，列表 CRUD 和评分入口 |
| `assessment_template` | 考核模板管理 | 后续页面 | 管理评分项和权重 |
| `assessment_indicator` | 考核指标管理 | 后续页面 | 管理专业能力、工作态度等指标 |
| `assessment_archive` | 考核结果归档 | 后续页面 | 查看历史批次归档 |

## 数据表

第一版建议 8 张表。菜单、基础主数据、考核记录、评分明细、操作审计拆开，避免把所有字段塞进一张野表。

| 表名 | 主要字段 | 职责 |
|---|---|---|
| `pal_menu_node` | `id,parent_id,menu_code,menu_name,page_code,icon,sort_no,enabled` | 左侧菜单树 |
| `pal_department` | `id,dept_code,dept_name,sort_no,enabled` | 部门字典 |
| `pal_mentor` | `id,mentor_code,mentor_name,dept_id,phone,enabled` | 导师字典 |
| `pal_intern_student` | `id,student_no,student_name,phone,dept_id,mentor_id,entry_date,enabled` | 实习生主数据 |
| `pal_assessment_batch` | `id,batch_code,batch_name,start_date,end_date,status` | 考核批次 |
| `pal_assessment_record` | `id,student_id,batch_id,mentor_id,professional_score,attitude_score,task_score,teamwork_score,innovation_score,total_score,status,comment,updated_at,deleted` | 列表主记录 |
| `pal_assessment_score_detail` | `id,record_id,item_code,item_name,weight,score,remark` | 评分弹窗明细 |
| `pal_operation_log` | `id,biz_type,biz_id,action,operator,message,created_at` | 审计日志 |

## 后端方法职责

所有方法统一返回：

```text
{ ok, code, message, data, traceId }
```

公共函数：

| 函数 | 职责 |
|---|---|
| `buildSuccess(data)` | 包装成功返回 |
| `buildFailure(code, message, traceId)` | 包装失败返回 |
| `handleException(error, context)` | 统一异常出口，写日志并转换业务错误码 |
| `assertRequired(value, fieldName)` | 必填校验 |
| `assertScoreRange(score, fieldName)` | 校验分数 0 到 100 |
| `assertRecordExists(record)` | 记录不存在时抛 `NOT_FOUND` |
| `writeOperationLog(action, bizId, message)` | 写审计日志 |

异常类型：

| 异常 | code | 场景 |
|---|---|---|
| `ValidationException` | `PAL_400` | 参数缺失、分数越界、日期非法 |
| `NotFoundException` | `PAL_404` | 学生、批次、考核记录不存在 |
| `ConflictException` | `PAL_409` | 重复学号、重复批次记录 |
| `DatabaseException` | `PAL_500_DB` | 数据读写失败 |
| `PermissionException` | `PAL_403` | 后续接入权限时使用 |

业务方法：

| 方法 | 入参 | 返回 | 职责 |
|---|---|---|---|
| `queryMenuTree()` | 无 | 菜单树 | 返回左侧菜单，不混入考核列表逻辑 |
| `queryAssessmentPage(filter)` | keyword、deptId、mentorId、batchId、status、dateRange、pageNo、pageSize | 分页列表 | 只查列表，不算复杂统计 |
| `queryDashboardStats(filter)` | 同筛选条件 | KPI 和图表数据 | 返回总数、待考核、已完成、平均分、部门均分、完成趋势 |
| `queryAssessmentDetail(recordId)` | recordId | 主记录 + 评分明细 | 弹窗打开前读取详情 |
| `createAssessmentRecord(payload)` | student、dept、mentor、batch、初始状态 | 新记录 | 新增考核记录，防重复 |
| `updateAssessmentRecord(id, payload)` | id + 可编辑字段 | 更新后记录 | 编辑基础信息，不负责评分提交 |
| `deleteAssessmentRecord(id)` | id | 删除结果 | 软删除，写日志 |
| `submitAssessmentScore(id, scorePayload)` | 五项分数 + 评语 | 更新后记录 | 校验分数、计算总分、写明细、状态改为 completed |

## 弹窗

弹窗建议独立页面/独立配置，命名 `PalimpsestScoreDialog`。弹窗只负责显示和提交评分，不直接改菜单和列表布局。

| 方法 | 入参 | 返回 |
|---|---|---|
| `openScoreDialog(recordId)` | recordId | `queryAssessmentDetail(recordId)` 的数据 |
| `saveScoreDialog(recordId, scorePayload)` | recordId + 五项分数 + 评语 | `submitAssessmentScore` 结果 |

综合评分：

```text
total_score =
  professional_score * 0.40 +
  attitude_score * 0.20 +
  task_score * 0.20 +
  teamwork_score * 0.10 +
  innovation_score * 0.10
```

分数保留 1 位小数。任一分项缺失时不允许提交评分。

## 实施顺序

1. 在后台模型建立基础假数据函数，先返回静态 JSON。
2. 建表或等价 WOS4 数据源对象准备完成后，把静态 JSON 改为表查询。
3. 先打通 `queryMenuTree / queryAssessmentPage / queryDashboardStats`。
4. 再做 `create/update/delete`。
5. 最后做 `submitAssessmentScore` 和弹窗。
6. 所有方法先走本地调试配置验证，再接入前端 `Call`。

## 停止条件

- 无法确认当前 WOS4 后台模型是否有可用数据源时，停止，不伪造数据库已落地。
