## 先理清的对象关系

当前已验证和只读参考都说明，WOS4 里至少有这几层：

```text
建模/页面设计阶段
-> 模型 / 页面精灵图 / 客户端页面列表

发布阶段
-> 提交版本
-> 生成拷贝
-> 仓库打包

运行阶段
-> 时空
-> 时空下挂仓库包
-> 部署 / 启动
-> 运行对象 / 客户端运行壳
```

关键区别：

- 页面不是在“时空里现场画”的。
- 页面先在建模/组态侧成为页面对象，再被客户端页面列表引用和挂载。
- 时空主要承载的是运行包、运行对象、时空范围和后端可见性，不是页面编辑入口。

## 页面与客户端关系

现有参考证据已经说明“客户端直接引用页面”这件事成立：

- `lk_客户端01` 的画面列表有：
  - `首页`

  - `画面引用器`

这说明：

1. 客户端有自己的页面列表和首页概念。
2. 页面可以通过客户端配置被挂到客户端里，不需要到时空里重新画一遍。
3. `框架页面` 很可能是承载页，真正业务页通过“添加画面/引用”方式进入客户端运行壳。

因此后续排查 `盛云_孙宇飞_客户端Demo_0617` 时，优先顺序不是再改画布，而是：

1. 客户端页面列表
2. 首页设置
3. 画面引用关系
4. 框架页是否正确承载业务页
5. 客户端更新版本后正式预览是否挂页

## 命名与脚本前缀规则

后续命名统一按三层处理：

1. 对外可见、会留在系统里的正式对象：
   - 使用 `盛云_孙宇飞_<业务语义>_<日期或版本>`

   - 可以使用非产品化、少见、带诗意的英文代号

   - 用中性前缀，如 `__wos4...`、`demo_...`、`temp_...`

```text
正式名写业务语义，临时代号写非产品化意象词，脚本前缀写中性技术词。
```

这条顺序里：

- “把东西放到时空里”发生在仓库包层和运行对象层。

- 这两件事有关联，但不是同一层动作。

原则：

- 如果动作可能污染公共环境、误改公共对象、触发部署/停止/删除，就必须先停。

## 分阶段预计情况与门槛

### 0. 会话与入口

预计情况：

- `browser-harness --doctor` 正常。
- 能进入主桌面。
- 双击客户端后会出现 `/public/` 和 `GetFileContent` 请求。
- 顶层 URL 可能不变，但 iframe 会变化。

关键小测试节点：

- `doctor` 通过。
- 登录后 `#/main` 可见。
- 目标客户端请求发出。
- iframe 中出现客户端关键文本。

关键小验收节点：

- 已进入目标客户端内部，不依赖动态 URL。
- 能稳定复读当前 frame 文本和当前上下文。

什么时候停止：

- 连续两次点击都没有任何目标网络请求。
- 登录态明显失效，页面回到登录或报未授权。
- 当前浏览器会话被公共弹窗或异常窗口覆盖，无法确认点击对象。

什么时候求助人类：

- 需要人工重新登录安全账号。
- 需要人工确认当前 Chrome 会话是否被别的同事占用。

### 1. 后端 FU 修改

预计情况：

- 能打开 `盛云_孙宇飞_元语言查询Demo_0617`。
- 能保存脚本。
- 能提交版本。
- 能生成新拷贝和新包。

关键小测试节点：

- 编辑器能重新打开已有 FU。
- 保存按钮有成功提示。
- 提交版本后版本号增加或历史版本列表变化。
- 生成拷贝后在列表里能看到新拷贝。

- 新版本 FU 的变更能在对应拷贝/包链路里被追到。
- 不需要新建第二个平行 FU。

什么时候停止：

- 保存成功但重开后脚本丢失。
- 提交版本连续两次失败且错误码一致。
- 拷贝或打包入口不再可见，或者对象被锁定。

什么时候求助人类：

- 出现人工审批/签名/电子签名弹窗。
- 需要确认是否允许覆盖已有包版本命名规则。

- 能打开 `盛云_孙宇飞_画面Demo_0617`。
- 页面结构仍是 `RContainer -> RRow -> RCol`。

什么时候停止：

- runtime 不存在。
- 页面重开后布局树损坏或对象 id 大面积漂移，导致无法继续精确修改。
- 连续两次提交后内容都不持久。

什么时候求助人类：

- 需要人工确认是否允许替换当前正式使用页面。
- 发现已有页面里存在业务上不能碰的脚本或配置，但文档里没有说明。

### 3. 客户端挂页关系

预计情况：

- `盛云_孙宇飞_客户端Demo_0617` 的问题重点在页面列表、首页、引用关系。
- `更新版本` 后正式预览才能代表真实客户端状态。

关键小测试节点：

- 能打开客户端页面列表。
- 能读到 `首页 / 画面名称 / 添加 / 画面引用器` 相关 UI。
- `更新版本` 成功提示可捕获。
- 正式预览链接可获取。

关键小验收节点：

- 客户端页面列表里能确认目标页面是否被引用。
- 首页或承载页配置明确，不再靠猜。
- 正式客户端壳中 `GetPageMngInfo()`、`showPageList` 或同类结构不为空。

什么时候停止：

- `alert` 继续报“页面不存在”，且页面版本映射没有变化证据。
- 正式预览壳仍为空，且客户端页面列表配置读不到更多有效信息。
- 当前操作可能把公共客户端首页切走。

什么时候求助人类：

- 需要人工确认哪一张页面应该是正式首页。
- 需要人工确认“画面引用器”的业务意图或已有约定。

### 4. 时空部署与运行对象

预计情况：

- 继续复用 `盛云_孙宇飞_时空_0617_TOPV2`。
- 新包应可添加、部署、启动。
- 运行对象应能继续在 `时空对象管理` 中创建或复用。

关键小测试节点：

- 时空可见。
- 包版本可见。
- 部署后状态变化为 `已部署`。
- 启动后状态变化为 `运行`。
- `onCreate` 或日志仍可见。

关键小验收节点：

- 后端运行对象正常。
- 当前链路不要求新建顶层时空。

什么时候停止：

- 现有时空状态异常，出现非预期停止、失败、锁定。
- 公共时空被选中而不是个人时空。
- 部署/启动按钮作用目标不清晰。

什么时候求助人类：

- 需要人工确认是否允许在现有时空上替换运行包。
- 需要人工确认某个时空是不是公共演示时空而非个人时空。

### 5. 前后端联调

预计情况：

- 只有在“页面已挂到客户端壳 + 后端对象已运行”后，`Call` 才有意义。
- 当前最大风险不是脚本语法，而是页面没挂对、时空没对齐、目标 App 不可见。

关键小测试节点：

- 页面正式壳非空。
- 后端对象运行中。
- `getLocalCloudID()` 可取值。
- 调用失败时能拿到 `ret/errorcodes/message`。

关键小验收节点：

- 至少一次 `Call` 不再返回 `-210133`。
- 按钮触发后页面数据有前后差异。

什么时候停止：

- 页面仍未挂载，却开始反复调按钮脚本。
- 同一个 `-210133` 连续复现且没有任何运行态变化。
- 无法确认当前前端实际连的是哪一个时空。

什么时候求助人类：

- 需要人工确认目标后端方法名、对象名、业务期望返回结构。
- 需要人工确认当前环境是否存在额外权限或访问时空配置。

## 全局停止条件

出现以下任一情况，当前轮次必须停止，不得继续盲试：

1. 连续两轮重复同一失败动作，没有新增证据。
2. 当前目标对象不再确定是个人对象。
3. 出现删除、停止、强制解锁、覆盖公共配置等高风险按钮，且目标不明确。
4. 页面、客户端、时空三层关系已经失焦，无法说明“我现在改的是哪一层”。
5. 需要靠猜测对象含义、首页职责、引用关系才能继续。

## 必须求助人类的情况

出现以下任一情况，应暂停并向人类确认：

1. 需要判断某个对象是不是正式生产/公共演示对象。
2. 需要决定某客户端哪一张页面应作为首页。
3. 需要电子签名、安全账号、额外审批。
4. 需要替换已有运行包，但无法确认是否影响他人。
5. 文档、手册、UI 三者互相冲突，且代码/运行证据无法裁决。
6. 已经完成两轮“客户端页面列表 / 首页 / 画面引用器 / 更新版本 / 正式预览”排查，仍无法判断哪一个字段才是正确挂页入口。
7. 需要在多个候选页面之间选择正式首页，但业务上无法从现有证据判断。

## Skill 分层整理原则

WOS4 skill 不是某一次操作录像，也不是固定坐标脚本。后续整理固定分三层：

```text
1. 入口和操作 skill
   登录、桌面客户端打开、iframe 识别、编辑器打开、保存、提交、预览。

2. 设计开发 skill
   需求 -> 布局树 -> RContainer/RRow/RCol -> 组件槽位 -> 变量 -> 交互 -> 样式 -> 验证。

3. 验证和维护 skill
   提交成功、重开持久化、预览可见、表格/图表高度、按钮/变量/脚本触发、版本更新和回归。
```

每个 skill 必须写清楚：

- 找什么业务文本。
- 进入哪个 iframe。
- 读取哪个 Vue/runtime 结构。
- 操作哪个平台对象或方法。
- 动作后验证什么状态。
- 失败时保存什么证据。

每个 skill 禁止写成：

- 固定屏幕坐标。
- 等待几秒后假设成功。
- 直接打开动态 `/public/` 或 `GetFileContent` URL。
- 只看截图就判定完成。

推荐数字员工工作方式：

```text
语义导航 + DOM 定位 + Vue runtime 操作 + 动态坐标真实点击 + 证据验证
```

能用运行时对象改，就不用鼠标拖；能用 DOM 文本定位，就不用截图识别；必须真实点击时，从元素矩形实时计算坐标；每一步都要有可读证据。

运行态版本维护的默认策略：

- 优先复用固定个人测试时空，不因每次改动新建时空。
- 更新模型后必须提交版本、生成拷贝、仓库打包，再进入运行态更新。
- 新建顶层时空只作为隔离验证或排障 fallback，不能作为常规开发流程。
- 如果已有运行时空的同仓库新包版本无法启动，先把失败边界写入 notes，再决定是否使用隔离时空。

## 手册推导出的部署运行主链路

手册 `快速入门` 给出的部署运行主链路是：

```text
时空功能开发
-> 新建源语言/元语言功能单元
-> 编译
-> 提交
-> 生成拷贝
-> 自启动 APP 配置
-> 打包
-> 部署管理
-> 时空对象管理
-> 新建时空对象
```

## 老师傅经验补充

以下内容来自人工经验，不是手册原文。后续只能作为操作假设和设计约束，必须经 UI/运行验证后才能沉淀为 skill。

- 数字孪生模型中新建内容时，前端和后端本质上可以放在同一个模型/功能体系里；为了可维护性，实际项目可以人为按业务拆成前端模块和后端模块。
- 前端低代码编辑入口应重点关注“页面精灵图”。页面精灵图可以理解为当前体系中的低代码前端编辑/页面设计载体。
- 页面里的脚本尽量放到“变量改变脚本”中，把按钮点击、打开时、定时任务等入口统一转换成“改变某个中转变量”来触发逻辑。
- 这种写法的目标是复用：按钮、页面打开、定时刷新都只负责改变量，真正业务脚本绑定在变量变化上，避免每个入口重复写一份逻辑。
- 这与现有页面设计 hard rule 兼容：先布局，再放组件，再配置变量和交互；变量联动优先用中转变量承接。

### 3. 运维部署客户端

手册依据：

- `系统客户端概述`：系统客户端管理区域、节点、资源组、进程等系统对象，并监控性能和报警。

- `资源组管理`：服务实例、数据区、资源上限、服务启停、数据区迁移。
- `区域管理`：云区域、传输网关和区域连接信息。
- `固定端对象管理`：固定端/微云接入、进程启停、硬件和性能监视。

验证假设：


它更像发布前后的环境检查和部署状态验证入口，不应首先用来新建页面。

安全约束：

- 只读查看节点、资源组、数据区、服务状态。
- 不执行启动、停止、重启、删除、卸载、迁移等操作，除非后续用户明确授权。

### 4. 时空对象管理

手册依据：

- `任务栏应用管理`：时空对象管理用于运行管理，包括时空对象、FU 对象、数据对象和记录。
- `应用图标管理`：FU 可以从桌面图标启动，也可以从时空对象管理“我创建的”或“全部”启动、停止、删除；WebJS FU 运行成功后可打开窗口。
- `模型库`：时空对象是业务应用的容器，包含 App 对象、数据对象和时空仓库。

验证假设：

```text
主桌面 / 任务栏
-> 时空对象管理
-> 我创建的 / 全部
-> 查看 FU/App 对象
-> 启动或打开已运行 WebJS FU
-> 验证页面窗口和运行状态
```

它是发布/部署后的运行验证入口，也是确认桌面图标、WebJS 窗口和 FU 状态的关键位置。

## 推荐下一步验证顺序

1. 不进 KingStudio。
2. 先从 `建模系统客户端_0518` 继续验证成员、提交版本、生成拷贝。
3. 进入 `时空功能开发` 的个人仓库，优先验证源语言/元语言 FU 的编辑、编译、提交、生成拷贝、自启动 APP 配置、打包、部署管理。

   最新实测已确认：
   - 正式预览链路已不再是旧 `mode=debugger` 页面。
   - 页面模型里 `spaceTime=盛云_孙宇飞_时空_0617` 已持久化。

  - 哪张页面应为正式首页

  - `框架页 -> 业务页` 的本项目约定到底是什么
5. 只读进入 `运维部署客户端_0518`，确认节点、资源组、数据区、服务状态。
