# WOS4 版本树

本文件描述 WOS4 平台对象之间的逻辑版本归属。它不是代码生成清单，也不是替代 WOS4 平台历史版本的本地源码树。

## 本地 Git 管理范围

- Git 根目录：`D:\DEV_D\WOS4.5`
- AI 规则和身份：`AGENTS.md`、`AI-INTERACTION-PANEL.md`、`MEMORY.md`、`.ai/agents/`、`.ai/instructions/`
- 项目 skill：`.ai/skills/`
- 项目设计和流程文档：`.ai/docs/`
- WOS4 可审计证据：`wos4-artifacts/backups/`、`snapshots/`、`screenshots/`、`tests/`、`reports/`
- 本地连接示例：`wos4-artifacts/config/*.example.ini`

真实账号、密码、Cookie、Token、`wos4.local.ini` 和运行时会话文件不进 Git。

## 引入规则

- `AGENTS.md` 引用本文件；涉及版本定位、模块归属、前后端联调或发布部署时，先读本文件。
- 具体操作仍以对应 skill 为准，例如布局用 `wos4-layout-devtools-skill`，发布用 `wos4-blue-client-publish-flow`，运维更新用 `wos4-runtime-package-update`。
- 后端链路的当前规范：业务事、自定义计算等模型内容在建模系统开发；运行实例在组态系统配置；部署、更新、启动在运维部署客户端完成；时空对象管理平台是已上线对象的时空信息获取和调试入口，不是新建入口。
- 每次修改前端布局、组件脚本、后端函数或发布链路时，备份目录必须记录实际 WOS4 路径、平台版本号、操作者和证据文件。
