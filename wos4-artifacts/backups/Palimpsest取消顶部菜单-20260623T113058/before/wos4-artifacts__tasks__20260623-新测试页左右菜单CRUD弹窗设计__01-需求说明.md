# 01 需求说明

## 用户原始描述

用户要求：

> 你是设计ai，而我们要再做一个全新的测试页，他应该有两部分组成，一个是左侧的菜单，右侧的页面。你需要去看看这个系统能实现让客户端上显示左右两个页面吗。然后需要准备假数据，的增删改查用的数据库，还有一个弹出页；先设计

用户追加确认：

> 代号"盛云科技_孙宇飞_Palimpsest"这是我选的代号，然后我明确说一下，大概率是可以让一个客户端左侧显示菜单页面，右侧内容页面的

用户提供参考图：

- `assets/参考图-人才管理考核页.png`

## 业务目标

设计一个新的 WOS4 管理类测试页，代号 `盛云科技_孙宇飞_Palimpsest`，用来验证：

- 客户端中左侧菜单页面 + 右侧业务内容页面的结构。
- 假数据列表、筛选、统计、图表和 CRUD 操作。
- 通过弹出页完成新增、编辑或评分。
- 后续可扩展到正式客户端蓝色 WebJS 对象中运行，并验证前端调用后端 CRUD 函数。

## 输入材料

- 图片：人才管理系统风格的考核管理页截图。
- 参考页面：已有 clean CRUD 页面 `查询列表` 的历史证据和技能记录。
- 现有 WOS4 路径：`组态系统客户端0518 -> 数字孪生可视化 -> 客户端/画面`，以及建模客户端内的页面精灵图编辑器。
- 现有后端/模型：历史 clean CRUD 后端 `DemoCrudBackend_CUSTOMFUNC` 已证明自定义函数可创建、保存和提交；正式按钮调用仍有上下文问题。
- 现有数据结构：已有 `QueryRecords` 假数据返回、`seedData/filterKey/refreshCounter` 页面变量模式。
- 用户期望利用的已有数据：本次先准备假数据和 CRUD 用数据结构，不要求立刻接真实生产数据。

## design-ai 方案核实

- 已确认存在的系统内容：
  - 页面内左右分栏布局能力已由 `wos4-layout-devtools-skill` 约束，可用 `RContainer/RRow/RCol` 实现。
  - 输入框、按钮、表格、折线图属于已验证组件。
  - 树组件曾出现但未在当前主线重新完成正式验收。
  - 客户端可在 `数字孪生可视化` 下添加/引用页面精灵图并设置首页。
  - 蓝色 WebJS 客户端对象创建和正式视图挂载已有历史可用路径。
  - 用户确认从业务判断看，客户端左侧菜单页面 + 右侧内容页面大概率可实现。
- 待核实的系统内容：
  - 同一客户端运行视口内，是否能同时显示两个独立页面对象。
  - WOS4 是否有稳定可持久化的弹窗组件，或是否需要用同页遮罩/抽屉模拟。
  - 表格行内按钮、行选择事件和弹窗回填路径是否能稳定保存。
  - 正式按钮 Click 脚本调用后端 CRUD 是否能在可见客户端中返回 `ret=0`。
- 缺口和风险：
  - 不能把“客户端挂载多个画面”直接等同于“左右两个页面同时显示”。
  - 当前按钮 Call 后端曾出现 `PageView.eval` 成功但可见按钮失败的边界，实施时必须专门验证执行上下文。
- 推荐方案：
  - 第一验证目标改为客户端双页面并排：左侧 `PalimpsestMenu`，右侧 `PalimpsestContent`。
  - 若双页面并排在 WOS4 正式客户端中不稳定，再降级为单页面左右分栏：左侧菜单 + 右侧内容区都在同一个页面精灵图中。
  - CRUD 先用假数据和后端函数契约设计，前端可先本地 mock，后续再替换为 `Call(...)`。
  - 弹窗先设计为同页弹出层；实施失败时降级为右侧抽屉。
- 阶段计划：
  - design-ai 完成方案。
  - frontend-ai 细化布局并创建页面。
  - code-ai 创建或复用后端 CRUD 函数。
  - test-ai 做预览、正式客户端和后端调用验收。

## 范围

本次要做：

- 设计新测试页的信息架构、布局、组件、交互、数据结构和验收标准。
- 创建任务工作包和协作事项。

本次不做：

- 不直接进入 WOS4 创建页面。
- 不修改线上页面、后端模型或运行包。
- 不声称“客户端双页面并排显示”已验证完成。

## 待确认问题

- 双页面并排的具体平台机制：是客户端布局支持、页面容器支持，还是需要页面跳转/嵌入能力配合。
- 弹出页是否必须是平台级弹窗，还是可以先用同页遮罩/抽屉模拟。
- CRUD 是否第一阶段必须真实持久化到 WOS4 数据模型，还是可以先用假数据函数返回并验证流程。
