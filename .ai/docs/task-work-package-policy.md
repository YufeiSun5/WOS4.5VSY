# 任务工作包规则

复杂任务必须使用“任务工作包”。交互面板只保留索引和状态，具体讨论、设计、执行日志、测试和证据放到任务目录里。

## 什么时候创建任务工作包

满足任一条件就创建：

- 用户给了图片、设计稿、截图或页面布局要求。
- 用户给了草图、前端交互说明、后端数据结构、已有数据利用需求，或需要先讨论方案。
- 任务包含前端布局、前端逻辑、后端改动、发布部署、测试验收中的两个以上部分。
- 任务预计需要分阶段完成。
- 任务需要多人或多个 AI 身份协作。
- 任务需要保存截图、快照、备份、报告或测试用例。

小型纯文档修改可以不建任务工作包，但仍要按备份规则操作。

## 放在哪里

任务工作包统一放：

```text
wos4-artifacts/tasks/
```

目录名优先中文，带日期：

```text
yyyyMMdd-中文任务名/
```

## 实施前门禁

复杂任务实施前运行：

```powershell
powershell -ExecutionPolicy Bypass -File wos4-artifacts/scripts/ai-preflight.ps1 -TaskPath wos4-artifacts/tasks/<任务目录>
```

涉及 WOS4 页面、模型、提交、预览、发布或运维部署时追加 `-Wos4`。preflight 失败时，先处理失败项，再继续实施。

## 标准结构

```text
任务目录/
├─ README.md
├─ 方案设计.md
├─ 01-需求说明.md
├─ 02-设计布局.md
├─ 03-前端逻辑.md
├─ 04-后端改动.md
├─ 05-测试验收.md
├─ 证据清单.md
├─ 执行日志.md
├─ assets/
├─ screenshots/
├─ snapshots/
└─ backups/
```

## 分阶段规则

一个任务可以分阶段完成。阶段写在任务 `README.md` 和 `执行日志.md` 中。

推荐阶段：

1. 需求澄清
2. 方案设计和系统核实
3. 布局设计
4. 前端逻辑
5. 后端改动
6. 联调验证
7. 回归测试
8. 关闭归档

## 交互面板怎么写

`AI-INTERACTION-PANEL.md` 只写简要索引：

- 状态
- 发起人
- 发起人 AI
- 参与人
- 任务工作包路径
- 改动范围
- 当前阶段
- 关闭条件

具体事件不要堆在交互面板，写到任务工作包的 `执行日志.md`。

## 图片、前端和后端

- 用户给的图片放 `assets/`。
- design-ai 先把需求方案、已有系统内容、数据结构核实和阶段计划写入 `方案设计.md`，必要的用户原始描述同步到 `01-需求说明.md`。
- 图片布局分析写到 `02-设计布局.md`。
- 前端页面变量、组件清单、按钮和交互写到 `03-前端逻辑.md`。
- 后端模型、函数、接口、入参返回和发布要求写到 `04-后端改动.md`。
- 测试计划写到 `05-测试验收.md`。
- 截图、快照、报告、备份和外部证据必须登记到 `证据清单.md`，不要只靠散落文件名猜上下文。

AI 根据用户描述创建任务工作包；用户可以直接修改任务文件，AI 后续按用户修改后的内容继续执行。
