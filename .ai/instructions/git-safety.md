# Git 安全控制

- Git 是 WOS4 自动化修改的安全控制方案。
- 修改前保留基线：页面模型、截图、关键脚本和测试计划。
- 修改后保留结果：页面模型、截图、验证 JSON、报告和 Git diff。
- 新建页面也要有基线记录：页面名、分组、配置摘要、创建前后截图、创建结果 JSON。
- 验证失败时保留失败证据，不覆盖现场。
- 敏感信息不得进入 Git。
- Git 提交记录必须使用中文，优先写清“做了什么”和“为什么”；必要的技术标识、远端名、分支名、文件名、函数名、平台对象名和错误码可以保留原文。
- 绑定多个远端时，GitHub 作为主维护远端使用 `origin`，Gitee 作为协作镜像远端使用 `gitee`；推送前必须先确认提交范围不含真实 ini、Cookie、Token、浏览器 profile 和抓包。
- 大量截图可后续考虑 Git LFS；当前先按目录规范保存。
